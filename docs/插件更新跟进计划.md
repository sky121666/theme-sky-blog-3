# 插件更新跟进计划

## 目的

这里记录最近更新过、且对当前主题有实际价值的 Halo 插件能力。

本文件只回答三个问题：

1. 哪些插件最近的版本值得跟进。
2. 这些更新对当前主题能带来什么功能收益。
3. 应该先验证什么，避免把“最新版本”误写成“主题已适配版本”。

契约版本仍以 [插件适配契约](插件适配契约.md) 为准。这里的“最新版本”只是跟进目标，不等于主题已经适配。

## 当前收口结论

当前主题已经完成的插件适配没有未知契约版本。继续跟进项只分三类：

1. 样本依赖复验：Docsme/Moments 的 Shiki、权限态、多语言多版本、富内容样本。
2. 可选增强：Equipment 是否进入组件中心、Photos 在有 tag/ungrouped 样本后是否做 REST 体验增强。
3. 长期观察：Douban 插件升级、封面代理说明或插件资源全局注入边界。

这些事项不阻塞当前版本发布；新增适配或增强时再按下方优先级执行。

执行层面的样本复验、性能采样和插件资源加载边界统一记录在 [验证与审计矩阵](验证与审计矩阵.md)。这里继续只保留插件版本和能力收益判断。

## 插件更新要点

### PluginMoments

当前跟进目标：`1.16.0`。

对主题有用的变化：

- `1.15.0` 提供新的公开 API，并完善模板引擎 Finder API。
- `1.12.0` 支持在瞬间列表显示点赞和评论数。
- `1.13.0` 支持 Shiki 代码高亮插件，`1.13.1` 修复 Shiki 重复渲染问题。
- `1.16.0` 使用 Halo 2.22 新索引 API，理论上提升查询性能和稳定性。

主题侧可做：

- `/moments` 列表模型、公开 API 和 Finder 字段已核对，主题继续优先消费路由注入和 Finder，不改为 headless REST。
- 点赞和评论数字段已统一，评论数优先使用 `approvedComment`，再回退 `totalComment`。
- PJAX 后 Shiki 已完成无代码块样本 smoke；已新增 `pnpm run verify:moments` 用于发现 Moments 代码块和已审核评论样例后做 PJAX 详情页复验。当前本地 Moments 评论样本已通过，代码块样本仍缺。
- RSS 依赖 `PluginFeed`，当前主题不把 Moments feed 标记为已适配能力；搜索类型 `moment.moment.halo.run` 暂由插件/全局搜索处理，不做专项结果 UI。
- [插件适配契约](插件适配契约.md) 中 `PluginMoments` 已确认到 `1.16.0`；后续真实 Shiki 代码块和已审核评论样本只影响专项复验，不影响 Moments 1.16.0 主题契约。

来源：

- Halo 应用市场：`https://www.halo.run/store/apps/app-SnwWD`
- 版本记录：`https://www.halo.run/store/apps/app-SnwWD/releases`

### PluginPhotos

当前跟进目标：`2.1.1`。

对主题有用的变化：

- 提供 `/photos` 列表路由和 `/photos/{name}` 详情路由。
- 支持名称、描述、链接、封面、标签、分组。
- 上传时自动读取 EXIF，包括相机、镜头、光圈、快门、ISO、焦距、图片尺寸。
- 提供匿名公共 REST API，可用于更动态的前端图库体验。
- `2.1.1` 将列表页和详情页部分模型变量改为惰性求值；主题正常 Thymeleaf 引用不需要改写。

主题侧可做：

- 保持详情页按 `PhotoVo`、`PhotoGroupVo`、`neighbors` 和 EXIF 数据继续完善。
- `/photos`、`/photos?page=1&size=20`、`/photos?view=albums`、分组页和详情入口已经通过本地 `PluginPhotos` 2.1.1 smoke，并已记录为 `confirmed`。
- 公共 REST API 已做只读评估，当前不切换实现：
  - `/apis/api.photo.halo.run/v1alpha1/photos?page=1&size=5` 返回 200。
  - `/apis/api.photo.halo.run/v1alpha1/photos?ungrouped=true` 返回 200，但当前本地样本为 0。
  - `/apis/api.photo.halo.run/v1alpha1/photos?tag=test` 返回 200，但当前本地没有 tag 样本。
  - `/apis/api.photo.halo.run/v1alpha1/photogroups` 返回 200。
  - `/apis/api.photo.halo.run/v1alpha1/photos/tags` 不是公共端点，返回 404。
- 本地样本统计：46 张照片、4 个分组、0 个未分组、0 个 tag、46 个 permalink、6 个 EXIF。
- 结论：当前 SSR/Finder + HTML 分页已经覆盖列表、分组、详情和 EXIF；公共 REST 适合未来做无刷新分页、tag 视图、ungrouped 入口或详情预取，但当前没有足够样本收益，不进入实现。

来源：

- Halo 应用市场：`https://www.halo.run/store/apps/app-BmQJW`
- 版本记录：`https://www.halo.run/store/apps/app-BmQJW/releases`

### plugin-shiki

当前跟进目标：`1.3.1`。

对主题有用的变化：

- `1.3.0` 升级 Shiki 至 4.0.0。
- `1.3.1` 修正末尾空代码行渲染，避免多余空行或行号显示。
- 支持在非文章页面启用代码高亮，例如瞬间、Docsme 文档页面。
- 支持暗色和亮色主题、代码行高亮、行聚焦、代码对比。

主题侧可做：

- 不重复接管代码块渲染。
- 在 PJAX 切页后只触发必要的插件生命周期，不重复渲染；主题已补偿 Shiki extra-path inline 脚本在 PJAX 后错过当前渲染时机的问题。
- 已验证文章页真实代码块、暗色标记和 extra-path replay 脚本清理；Docsme、Moments 后续需要补真实代码块内容样本后再做最终确认。

来源：

- Halo 应用市场版本：`https://www.halo.run/store/apps/app-kzloktzn/releases/app-release-vru08juv`

### plugin-docsme

当前跟进目标：`1.6.0`。

对主题有用的变化：

- 支持多项目、多语言、多版本。
- 支持评论。
- 支持文档项目权限。
- Docsme `1.6.0` 主题覆盖时需要保留 `_templateId`，并优先消费插件注入的文档描述 meta。

主题侧结论：

- 已确认 `_templateId` 没有被主题硬编码替代，文档页输出 `data-docsme-template-id="plugin:docsme:doc"`。
- 已确认文档描述能进入页面 SEO，本地可访问文档详情页输出文档级 `meta description`。
- 已确认评论容器、目录、同 App PJAX 协议字段正常。
- 权限态、多语言多版本切换、Shiki/KaTeX/Mermaid 真实内容样本仍依赖后续本地数据补齐；已新增 `pnpm run verify:docsme`，有样本时自动复验，没有样本时明确跳过。

来源：

- 官方 Docsme 文档：`https://www.halo.run/docs/docsme`

### equipment

当前跟进目标：`1.1.1`。

对主题有用的变化：

- 应用市场显示当前版本 `1.1.1`。
- 插件提供 `/equipments` 页面路由，内置模板，也支持主题自定义模板。
- 技能记录显示 `1.1.1` 有匿名公开 REST 查询接口。

主题侧可做：

- `1.1.1` 新增的公开 API 适合 headless 页面或外部组件，不作为当前主题必要适配。
- 契约版本记录为 `1.1.0`，已测试版本记录为 `1.1.1`，不把公开 API 增量倒填成主题契约。

来源：

### plugin-douban

当前跟进目标：`1.2.5`。

对主题有用的变化：

- `1.2.5` 新增 types 接口：`/apis/api.douban.moony.la/v1alpha1/doubanmovies/-/types`。
- types 返回分类 key、名称和数量，适合做豆瓣页筛选导航。

主题侧结论：

- `/douban` 独立 App 已新增，顶部类型筛选优先使用 `types` 接口，题材筛选使用 `genres` 接口。
- 当前实现采用主题自定义 UI 和公开 API 渲染，不复用插件默认 `modules/douban` 片段。
- 当前 `1.2.5` 契约已确认；真实数据下筛选、搜索、分页、封面和空状态已由用户手动确认。
- 后续只在插件版本升级、封面代理配置需要说明或体验增强时继续跟进。

来源：

- Halo 应用市场版本：`https://www.halo.run/store/apps/app-srBOL/releases/app-release-ypwb3h7s`

## 验证顺序

1. `PluginMoments`：已确认到 `1.16.0`，后续只补真实 Shiki 代码块和已审核评论内容样本；有样例后运行 `pnpm run verify:moments`。
2. `PluginPhotos`：契约已确认到 `2.1.1`；公共 REST/tag/ungrouped 已评估，当前不做增强实现，后续只有补 tag/ungrouped 真实样本或要做无刷新分页时再打开。
3. `plugin-shiki`：补 Docsme/Moments 真实代码块样本后，再做 PJAX 重复渲染检查。
4. `plugin-docsme`：已补 SEO、评论和目录验证；后续只补权限态、多语言多版本和富内容样本。
5. `equipment`：已记录 1.1.1 字段和公开 API；当前主题继续走 SSR/PJAX 主链路，不改实现。
6. `plugin-douban`：契约已确认到 `1.2.5`；后续只按插件升级、封面代理说明或体验增强重新评估。

## 完成标准

每个插件完成跟进时，至少更新三处：

1. [插件适配契约](插件适配契约.md)：记录契约版本、已测试版本、状态和依据。
2. 对应模板的 `plugin-contract` 注释：只写确认、兼容测试或推断状态。
3. 对应功能文档：记录主题实际消费了哪些字段、路由和生命周期。
