# 插件更新跟进计划

## 目的

这里记录最近更新过、且对当前主题有实际价值的 Halo 插件能力。

本文件只回答三个问题：

1. 哪些插件最近的版本值得跟进。
2. 这些更新对当前主题能带来什么功能收益。
3. 应该先验证什么，避免把“最新版本”误写成“主题已适配版本”。

契约版本仍以 [插件适配契约](插件适配契约.md) 为准。这里的“最新版本”只是跟进目标，不等于主题已经适配。

## 优先级结论

| 优先级 | 插件 | 跟进目标 | 当前判断 |
| --- | --- | --- | --- |
| P0 | `PluginMoments` | 补验 `1.15.0+` 的公开 API、Finder、点赞/评论数、Shiki 内容兼容 | 已补公开 API、统计字段和 Shiki/PJAX smoke；含代码块内容样本待补 |
| P0 | `PluginPhotos` | 继续评估 2.1.0 公共 REST、tag、ungrouped 对主题的实际收益 | 主题契约已确认到 2.1.0；公共 REST/tag/ungrouped 后续只作为增强项 |
| P1 | `plugin-shiki` | 补 Docsme、Moments 真实代码块样本，复验 extra-path 渲染结果 | 文章页真实代码块和暗色标记已复验；Docsme/Moments 当前只确认 extra-path replay 脚本存在且不残留临时脚本 |
| P1 | `plugin-docsme` | 复查 `_templateId`、文档描述 meta、评论、权限态、Shiki/KaTeX/Mermaid 组合 | 契约已确认 `1.6.0`，属于质量补完 |
| P2 | `equipment` | 记录 `1.1.1` 公开 REST API，不接入主题主链路 | 主题契约按 `1.1.0`，本地 `1.1.1` 兼容；新增公开 API 对 SSR/PJAX 装备页不是必要适配 |
| P2 | `plugin-douban` | 如要新增 `/douban`，优先使用 `1.2.5` types 接口做分类筛选 | 当前主题未适配，需单独立项 |

## 插件更新要点

### PluginMoments

当前跟进目标：`1.16.0`。

对主题有用的变化：

- `1.15.0` 提供新的公开 API，并完善模板引擎 Finder API。
- `1.12.0` 支持在瞬间列表显示点赞和评论数。
- `1.13.0` 支持 Shiki 代码高亮插件，`1.13.1` 修复 Shiki 重复渲染问题。
- `1.16.0` 使用 Halo 2.22 新索引 API，理论上提升查询性能和稳定性。

主题侧可做：

- 确认 `/moments` 列表模型是否已经包含点赞数、评论数。
- 若 Finder 已完善，优先改用 Finder 提供的稳定字段，不在前端重复推断。
- 补 PJAX 后 Shiki 只初始化一次的验证；当前已完成无代码块样本 smoke，仍需补一条 Moments/Docsme 代码块内容样本。
- 公开 API 和统计字段通过后，把 [插件适配契约](插件适配契约.md) 中 `PluginMoments` 更新为 `compatible-tested`；Shiki/PJAX 全量通过后再考虑升为新的 `confirmed`。

来源：

- Halo 应用市场：`https://www.halo.run/store/apps/app-SnwWD`
- 版本记录：`https://www.halo.run/store/apps/app-SnwWD/releases`

### PluginPhotos

当前跟进目标：`2.1.0`。

对主题有用的变化：

- 提供 `/photos` 列表路由和 `/photos/{name}` 详情路由。
- 支持名称、描述、链接、封面、标签、分组。
- 上传时自动读取 EXIF，包括相机、镜头、光圈、快门、ISO、焦距、图片尺寸。
- 提供匿名公共 REST API，可用于更动态的前端图库体验。

主题侧可做：

- 保持详情页按 `PhotoVo`、`PhotoGroupVo`、`neighbors` 和 EXIF 数据继续完善。
- `/photos`、`/photos?page=1&size=20`、`/photos?view=albums`、分组页和详情入口已经通过本地 `PluginPhotos` 2.1.0 smoke，并已按用户确认记录为 `confirmed`。
- 验证公共 REST API 是否适合后续做无刷新分页、相似照片、详情预取。
- 公共 REST API、tag、ungrouped 和详情预取属于后续增强，不影响当前 2.1.0 主题契约确认。

来源：

- Halo 应用市场：`https://www.halo.run/store/apps/app-BmQJW`
- 版本记录：`https://www.halo.run/store/apps/app-BmQJW/releases`

### plugin-shiki

当前跟进目标：`1.3.0`。

对主题有用的变化：

- `1.3.0` 升级 Shiki 至 4.0.0。
- 支持在非文章页面启用代码高亮，例如瞬间、Docsme 文档页面。
- 支持暗色和亮色主题、代码行高亮、行聚焦、代码对比。

主题侧可做：

- 不重复接管代码块渲染。
- 在 PJAX 切页后只触发必要的插件生命周期，不重复渲染；主题已补偿 Shiki extra-path inline 脚本在 PJAX 后错过当前渲染时机的问题。
- 已验证文章页真实代码块、暗色标记和 extra-path replay 脚本清理；Docsme、Moments 后续需要补真实代码块内容样本后再做最终确认。

来源：

- Halo 应用市场版本：`https://www.halo.run/store/apps/app-kzloktzn/releases/app-release-vru08juv`

### plugin-docsme

当前跟进目标：`1.6.0`。

对主题有用的变化：

- 支持多项目、多语言、多版本。
- 支持评论。
- 支持文档项目权限。
- Docsme `1.6.0` 主题覆盖时需要保留 `_templateId`，并优先消费插件注入的文档描述 meta。

主题侧可做：

- 检查 `_templateId` 没有被主题硬编码替代。
- 检查文档描述是否能进入页面 SEO。
- 验证评论、权限态、目录和语言/版本切换在 PJAX 后稳定。

来源：

- 官方 Docsme 文档：`https://www.halo.run/docs/docsme`

### equipment

当前跟进目标：`1.1.1`。

对主题有用的变化：

- 应用市场显示当前版本 `1.1.1`。
- 插件提供 `/equipments` 页面路由，内置模板，也支持主题自定义模板。
- 技能记录显示 `1.1.1` 有匿名公开 REST 查询接口。

主题侧可做：

- 当前主题继续使用 `/equipments` SSR 路由模型、`equipmentFinder` 和 HTML 下一页加载。
- `1.1.1` 新增的公开 API 适合 headless 页面或外部组件，不作为当前主题必要适配。
- 契约版本记录为 `1.1.0`，已测试版本记录为 `1.1.1`，不把公开 API 增量倒填成主题契约。

来源：

- Halo 应用市场：`https://www.halo.run/store/apps/app-ytygyqml`

### plugin-douban

当前跟进目标：`1.2.5`。

对主题有用的变化：

- `1.2.5` 新增 types 接口：`/apis/api.douban.moony.la/v1alpha1/doubanmovies/-/types`。
- types 返回分类 key、名称和数量，适合做豆瓣页筛选导航。

主题侧可做：

- 新增 `/douban` 独立 App 时，优先用 types 接口构建顶部筛选。
- 不把 Douban 混进当前已完成插件的兼容修复批次。

来源：

- Halo 应用市场版本：`https://www.halo.run/store/apps/app-srBOL/releases/app-release-ypwb3h7s`

## 验证顺序

1. `PluginMoments`：先验字段和 Shiki 兼容，因为它可能影响 feed、详情和搜索。
2. `PluginPhotos`：契约已确认到 `2.1.0`；后续只验证公共 REST/tag/ungrouped 是否值得做增强。
3. `plugin-shiki`：补 Docsme/Moments 真实代码块样本后，再做 PJAX 重复渲染检查。
4. `plugin-docsme`：补 SEO、权限态和评论验证。
5. `equipment`：验 1.1.1 字段和 API，再决定是否改实现。
6. `plugin-douban`：单独开新页面适配，不并入兼容修复。

## 完成标准

每个插件完成跟进时，至少更新三处：

1. [插件适配契约](插件适配契约.md)：记录契约版本、已测试版本、状态和依据。
2. 对应模板的 `plugin-contract` 注释：只写确认、兼容测试或推断状态。
3. 对应功能文档：记录主题实际消费了哪些字段、路由和生命周期。
