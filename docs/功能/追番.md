# 追番

## 结论

`/bangumis` 现在是独立 App，不复用通用 browser 窗口。

- `pageApp = bangumis`
- `pageMode = browser-bangumis`
- `windowVariant = bangumis`

数据继续来自 `plugin-bilibili-bangumi` 1.4.0 路由契约注入的 `bangumis` 模型，已在最新稳定版 1.4.1 上完成兼容回归；主题只消费插件结果，不重新请求 Bilibili。

当前契约版本为 `1.4.0`，已测试版本为 `1.4.1`，状态为 `compatible-tested`。

## 数据契约

支持插件原生路由：

- `/bangumis`
- `/bangumis/page/{page}`

支持插件原生参数：

- `typeNum`: `1` 追番，`2` 追剧
- `status`: `0` 全部，`1` 想看，`2` 在看，`3` 已看
- `page`
- `size`

模板读取 `bangumis.items`、`bangumis.total`、`bangumis.page`、`bangumis.totalPages`、`bangumis.prevUrl`、`bangumis.nextUrl`。

## 交互

左侧边栏提供类型和状态筛选，全部走 PJAX 链接。
主区域使用封面列表卡片，展示评分、集数、类型、地区、简介和播放/弹幕/追番/投币指标。
分页使用滚动加载模式：触底后 fetch 插件下一页 HTML，只追加新的列表卡片，不替换当前窗口。
搜索只过滤已加载条目，不改变插件分页结果。

## 移动端

移动端隐藏顶栏、Dock、交通灯和 resize handle，窗口铺满屏幕。
侧边栏在顶部横向滚动，内容区保持独立滚动。

## 验证

本轮补验覆盖：

- 主题契约固定为 `1.4.0`，模板标记已测试 `1.4.1`。
- `/bangumis` 返回 200，输出 `data-page-mode="browser-bangumis"`、`data-app-id="bangumis"`、`data-window-variant="bangumis"`。
- `/bangumis?typeNum=2&status=0` 和 `/bangumis?typeNum=1&status=2` 返回 200，类型/状态筛选模型正常。
- 浏览器验证 `/bangumis` 首屏 12 张卡片、搜索框、下一页 URL `/bangumis/page/2?typeNum=1&status=0` 正常。
- 浏览器验证 `/bangumis/page/not-a-number` 返回 404，适配 1.4.1 的非法页码行为。
