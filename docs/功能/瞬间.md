- `pageApp = moments`
- `pageMode = browser-moments`

## 后台联动

瞬间页主要依赖这些后台设置：

- `moments.cover.image_url`
- `moments.profile.display_name`
- `moments.profile.subtitle`

说明：

- 列表页和详情页都会受封面配置影响
- 留空时会回退到站点标题和副标题

## 关键行为

1. 列表 / 详情共用一个窗口体系
2. 标题栏会跟场景切换
3. `back / forward` 现在会正确恢复标题
4. 同一功能内的跳转依然通过公共 PJAX 链路完成
5. 列表页、作者页和桌面小组件消费 `MomentVo.stats.upvote`、`stats.approvedComment` / `stats.totalComment`，评论数统一优先使用已审核评论数
6. 列表页媒体类型覆盖 `PHOTO`、`VIDEO`、`AUDIO`、`POST`，详情页继续按插件媒体类型渲染
7. PJAX 进入 Moments 后会触发 Shiki extra-path replay；当前 smoke 已确认无重复脚本和无重复 `shiki-code`，但本地 Moments 内容暂缺代码块样本

## 专项验证

新增 Moments 样例验证命令：

```bash
pnpm run verify:moments
```

验证内容：

- 从公开 API `/apis/api.moment.halo.run/v1alpha1/moments` 自动发现 Moments 内容。
- 固定验证列表 API、详情 API 和主题依赖字段：`metadata.name`、`spec.content`、`stats.approvedComment`、`stats.totalComment`、`stats.upvote`。
- 如发现代码块样例，会先进入 `/moments`，再通过 PJAX 点击到详情页，检查页面协议、详情根节点、Shiki/代码块 DOM 和临时 replay 脚本清理。
- 如发现 `stats.approvedComment > 0` 的样例，会同样通过 PJAX 进入详情页，检查页面协议和评论容器。
- 结果写入 `output/playwright/moments-report.json`。

当前本地内容没有 Moments 代码块样例，也没有已审核评论样例，所以 `pnpm run verify:moments` 会返回 `1 passed, 2 skipped`。`passed` 是公开 API 契约测试，`skipped` 是站点内容缺少可验证样本。

补齐内容后可以显式指定样例路径复验：

```bash
MOMENTS_CODE_SAMPLE_PATH=/moments/<name> pnpm run verify:moments
MOMENTS_COMMENT_SAMPLE_PATH=/moments/<name> pnpm run verify:moments
```

## 前端发布

瞬间标题栏右侧提供相机按钮，用于登录用户在前端发布瞬间。

主题设置路径：

- `瞬间 -> 前端发布 -> 启用前端发布`
- 可单独控制图片、视频、音频上传入口。

接口边界：

```text
GET  /apis/api.console.halo.run/v1alpha1/users/-
POST /apis/uc.api.storage.halo.run/v1alpha1/attachments/-/upload
POST /apis/moment.halo.run/v1alpha1/moments
```

行为边界：

- 发布前先通过 `users/-` 解析当前登录用户，未登录时只显示提示，不写 PAT。
- 发布入口默认隐藏，只有 `users/-` 确认当前为登录用户后才显示。
- 媒体文件走 Halo UC 附件上传接口，发布 payload 的 `spec.content.medium` 使用 `PHOTO` / `VIDEO` / `AUDIO`。
- 标签写入插件原生字段 `spec.tags`，前端限制最多 10 个、单个最多 50 字符。
- 发布成功后通过 PJAX 回到 `/moments` 刷新列表。
- 与 `theme-sky-blog-1` 一致，Alist 存储不作为推荐上传后端；建议使用本地存储或 S3。

当前只适配插件和 Halo 用户态 API 能真实支撑的能力：

| 能力 | 状态 | 主题处理 |
| --- | --- | --- |
| 文字内容 | 支持 | 写入 `spec.content.raw` / `html` |
| 图片、GIF、视频、音频 | 支持 | 先上传到 Halo 附件，再写入 `spec.content.medium` |
| 标签 | 支持 | 写入 `spec.tags` |
| 常用 emoji | 支持 | 插入到正文，随文本一起发布 |
| 公开可见 | 支持 | 固定 `spec.visible = PUBLIC` |
| 提醒谁看 | 不支持 | 不显示入口 |
| 谁可以看 / 私密可见 | 不支持 | 不显示入口 |
| 定位、完整表情面板 | 未适配 | 不显示入口 |

## 顶部互动通知

Moments 窗口标题栏右侧有瞬间互动通知入口，只服务当前登录用户。

主题侧只接用户态通知接口，不接后台原始通知资源 API，也不使用 PAT。当前按 Halo OpenAPI 暴露的用户通知链路实现：

```text
GET /apis/api.console.halo.run/v1alpha1/users/-
GET /apis/api.notification.halo.run/v1alpha1/userspaces/{username}/notifications?page=1&size=50&sort=metadata.creationTimestamp,desc
```

行为边界：

- 先通过 `users/-` 获取当前登录用户的 `user.metadata.name`。
- 未登录或返回 `anonymousUser` 时显示“登录后查看瞬间互动消息”。
- 再用真实用户名请求 `api.notification` 用户空间通知，不能用 `userspaces/-` 占位符。
- 模板保留 `data-endpoint="/apis/api.console.halo.run/v1alpha1/users/-"` 作为旧版 JS 缓存兼容入口，新逻辑读取 `data-notifications-endpoint`。
- 请求必须使用浏览器登录态，前端 `fetch` 带 `credentials: 'same-origin'`，不在主题里写 PAT。
- 本地 OpenAPI 已确认 `new-comment-on-moment` reason type 存在，但 `Notification.spec.reason` 存的是 `reason-xxx` 资源名，不是 reason type。
- 主题先按通知标题、正文和链接保守筛选“瞬间 / moment / /moments”相关通知，避免把文章评论通知混入。
- `401` / `403` / 登录跳转时显示“登录后查看瞬间互动消息”。
- `404` 时显示“当前 Halo 未开放用户通知接口”，保留 UI，等待后续用户态 API 或插件 API 接入。
- 详情页和列表页共用同一个 titlebar 通知按钮；PJAX 切页后会重新绑定。

## 列表点赞和评论

列表页提供微信朋友圈风格的互动层。右下角 `··` 继续负责显示 `赞 / 评论` 操作；已有评论的瞬间会在可见区域附近自动渲染评论预览，但评论输入框默认隐藏。

接口边界：

```text
POST /apis/api.halo.run/v1alpha1/trackers/upvote
GET  /apis/api.halo.run/v1alpha1/comments
POST /apis/api.halo.run/v1alpha1/comments
```

点赞 payload 固定使用 Moments subject：

```json
{
  "group": "moment.halo.run",
  "plural": "moments",
  "name": "moment-xxx"
}
```

评论列表懒加载参数：

```text
group=moment.halo.run
version=v1alpha1
kind=Moment
name={momentName}
withReplies=true
size=5
replySize=2
```

行为边界：

- 列表页只对 `stats.approvedComment / totalComment > 0` 的卡片自动渲染评论预览。
- 自动评论预览使用 `IntersectionObserver`，卡片进入视口附近才请求；并发限制为 3，避免首页和无限滚动时请求爆发。
- 列表评论预览默认每页 5 条评论、每条评论带 2 条回复；点击“查看全部评论”进入 `/moments/{name}#moment-comments`，不在列表页继续分页。
- 详情页评论列表才继续提供“查看更多评论”分页加载。
- 访问 `/moments/{name}#moment-comments` 会自动展开详情评论区、滚动到评论区并做短暂高亮；预留 `#comment-{name}` 定位到具体评论的前端能力。
- 评论输入框默认隐藏；点击单条瞬间的“评论”或点击某条评论/回复时才展开并聚焦。
- 评论面板支持评论分页、回复分页、评论提交、回复提交、回复上下文提示和失败兜底；提交完成后输入框收起。
- 列表页发布时间、图片、视频、音频和关联内容占位进入瞬间详情；头像和昵称在 `ownerName` 存在时链接到 `/authors/{ownerName}`。
- 评论视觉按微信风格固定：用户名使用蓝灰色，绿色只用于发送按钮、输入框 focus 和回复态细线，不跟随主题大面积变色。
- 详情页使用同一套主题自渲染评论层，首屏自动加载当前瞬间评论；官方 `<halo:comment group="moment.halo.run" kind="Moment" />` 只作为评论 API 不可用时的隐藏兼容兜底。
- 不做取消点赞；前端用 `localStorage` 记录已赞，避免重复请求。
- 不显示点赞用户头像或昵称列表；当前 Moments 公开数据只提供 `stats.upvote` 数量，`trackers/upvote` 也只有写入接口，没有可查询点赞用户的公开 API。
- 评论区只展示真实评论数据；点赞区只展示心形和数量，不展示评论数量，避免把不同能力混在同一行。
- 不在列表页重写删除、审核和管理能力。
- 评论提交失败时保留输入内容；评论 API 不可用时才显示“打开详情页评论”的兜底入口。

## 瞬间配色模式

后台设置路径：

- `瞬间 -> 界面风格 -> 配色模式`

可选值：

| 模式 | 行为 |
| --- | --- |
| 微信风格 | 默认值。使用固定朋友圈拟态配色，交互主色固定为微信绿。 |

## PluginMoments 1.16.0 适配边界

主题契约按最新正式版 `PluginMoments` `1.16.0` 记录。主题适配范围是 Moments 前台路由、Finder/API 字段、统计字段、评论 subject、tag 筛选、媒体类型和 PJAX 生命周期；RSS/Feed 与搜索结果专项 UI 属于独立能力，不作为当前 Moments App 契约阻塞项。

已验证：

- 本地安装 `PluginMoments` `1.16.0`，插件状态 `STARTED`。
- 公开 API `/apis/api.moment.halo.run/v1alpha1/moments` 返回 200，字段包含 `stats.upvote`、`stats.approvedComment`、`stats.totalComment`、`owner`、`spec.tags` 和 `spec.content.medium`。
- `/moments`、`/moments/page/2`、`/moments/{name}` 返回 200，协议字段保持 `browser-moments` / `moments`。
- `tag` 查询参数使用插件路由和公开 API，中文标签需要 URL encode 后访问。
- 详情页评论使用 Moments 正确 subject：`group="moment.halo.run"`、`kind="Moment"`。

未承诺：

- RSS/Feed 依赖 `PluginFeed`，当前本地 `/feed/moments/rss.xml` 和 `/moments/rss.xml` 返回 404，主题不把 Moments feed 标记为已适配能力。
- 搜索类型 `moment.moment.halo.run` 由插件/搜索索引提供，当前主题只接入全局搜索入口，没有做 Moments 搜索结果专项 UI。
- Shiki 真实代码块需要补 Moments 内容样本后再复验；当前只确认 PJAX 后无重复临时脚本和无重复 `shiki-code`。这属于 Shiki extra-path 复验缺口，不阻塞 Moments 1.16.0 主题契约确认。
