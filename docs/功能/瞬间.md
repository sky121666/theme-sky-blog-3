# 瞬间

## 结论

瞬间是独立 App，协议是：

- `pageApp = moments`
- `pageMode = browser-moments`
- `windowVariant = moments`

列表页和详情页共享同一个功能目录，不再分散在旧的 `page/moments` 路径里。

## 后台联动

瞬间页主要依赖这些后台设置：

- `moments.cover.image_url`
- `moments.profile.display_name`
- `moments.profile.subtitle`

说明：

- 列表页和详情页都会受封面配置影响
- 留空时会回退到站点标题和副标题

## 关键行为

1. 列表 / 详情共用一个窗口体系
2. 标题栏会跟场景切换
3. `back / forward` 现在会正确恢复标题
4. 同一功能内的跳转依然通过公共 PJAX 链路完成
5. 列表页、作者页和桌面小组件消费 `MomentVo.stats.upvote`、`stats.approvedComment` / `stats.totalComment`，评论数统一优先使用已审核评论数
6. 列表页媒体类型覆盖 `PHOTO`、`VIDEO`、`AUDIO`、`POST`，详情页继续按插件媒体类型渲染
7. PJAX 进入 Moments 后会触发 Shiki extra-path replay；当前 smoke 已确认无重复脚本和无重复 `shiki-code`，但本地 Moments 内容暂缺代码块样本

## PluginMoments 1.16.0 适配边界

主题契约按最新正式版 `PluginMoments` `1.16.0` 记录。主题适配范围是 Moments 前台路由、Finder/API 字段、统计字段、评论 subject、tag 筛选、媒体类型和 PJAX 生命周期；RSS/Feed 与搜索结果专项 UI 属于独立能力，不作为当前 Moments App 契约阻塞项。

已验证：

- 本地安装 `PluginMoments` `1.16.0`，插件状态 `STARTED`。
- 公开 API `/apis/api.moment.halo.run/v1alpha1/moments` 返回 200，字段包含 `stats.upvote`、`stats.approvedComment`、`stats.totalComment`、`owner`、`spec.tags` 和 `spec.content.medium`。
- `/moments`、`/moments/page/2`、`/moments/{name}` 返回 200，协议字段保持 `browser-moments` / `moments`。
- `tag` 查询参数使用插件路由和公开 API，中文标签需要 URL encode 后访问。
- 详情页评论使用 Moments 正确 subject：`group="moment.halo.run"`、`kind="Moment"`。

未承诺：

- RSS/Feed 依赖 `PluginFeed`，当前本地 `/feed/moments/rss.xml` 和 `/moments/rss.xml` 返回 404，主题不把 Moments feed 标记为已适配能力。
- 搜索类型 `moment.moment.halo.run` 由插件/搜索索引提供，当前主题只接入全局搜索入口，没有做 Moments 搜索结果专项 UI。
- Shiki 真实代码块需要补 Moments 内容样本后再复验；当前只确认 PJAX 后无重复临时脚本和无重复 `shiki-code`。这属于 Shiki extra-path 复验缺口，不阻塞 Moments 1.16.0 主题契约确认。
