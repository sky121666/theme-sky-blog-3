# 桌面壳层

## 结论

桌面壳层是当前主题的公共运行底座。
它负责：

- 顶栏
- Dock
- 桌面图标
- 桌面小组件
- 主窗口
- PJAX

如果一个改动会影响多个页面，优先看这里。

## 当前职责边界

### 壳层负责

- shell HTML 结构
- 全局主题状态
- PJAX 内容替换
- 桌面层交互
- 浏览器窗口的统一行为

### 壳层不负责

- feature 页面正文细节
- 图库内容布局
- 瞬间内容布局
- explorer/reader 正文样式

这些都应该归对应 feature 目录。

## PJAX 原则

PJAX 只替换：

- `#window-frame-root` 内部内容

不替换：

- Header
- Dock
- Desktop Surface
- 主窗口壳层外框

当前相关实现：

- [/src/shell/desktop-shell/runtime/desktop/pjax/index.js](/src/shell/desktop-shell/runtime/desktop/pjax/index.js)
- [/src/shell/desktop-shell/runtime/desktop/pjax/css-router.js](/src/shell/desktop-shell/runtime/desktop/pjax/css-router.js)
- [/src/shell/desktop-shell/runtime/desktop/pjax/protocol.js](/src/shell/desktop-shell/runtime/desktop/pjax/protocol.js)

## 什么时候改这里

适合改壳层的需求：

1. 顶栏按钮行为
2. Dock 交互
3. 主窗口标题栏
4. 浏览器返回/前进逻辑
5. PJAX 同页切换
6. 桌面层编辑态

如果只是页面正文布局，不要先改 shell。

## 我想配什么，先配哪里

这些需求优先去后台设置，不要先改代码：

1. 顶栏 / Dock 菜单来源：看 `navigation.header.menu_name` 和 `navigation.dock.menu_name`
2. 顶栏标题和图标：看 `header.logo.*`
3. Dock 外观和交互参数：看 `dock.*`
4. 开发日志开关：看 `developer.debug_mode`

## 菜单位置兼容规则

Halo 后台菜单是一棵通用导航树，主题不会把它自动改造成不同位置的专用字段。不同位置只按自己的能力消费同一份菜单数据。

| 位置 | 配置入口 | 支持层级 | 消费规则 |
| --- | --- | --- | --- |
| 顶栏桌面端 | `navigation.header.menu_name` | 一级 + 二级 | 一级显示为顶栏文本；有二级菜单的一级项作为分组入口，不直接跳转。 |
| 顶栏移动端 | `navigation.header.menu_name` | 一级 + 二级 | 无二级的菜单项直接显示；有二级的菜单项按分组展开。 |
| Dock | `navigation.dock.menu_name` | 一级 | 只把一级菜单项渲染为 Dock 图标；二级菜单不会展开。 |
| 页面侧边栏 | 页面业务数据 | 由页面决定 | 通常来自分类、标签、相册、友链分组、Docsme 目录等，不等同于全局菜单。 |
| Footer | `<halo:footer />` | 非菜单体系 | 由 Halo 或插件注入，不从主题菜单设置读取。 |

兼容说明：旧版本保存过的 `header.source.menu_name` 和 `dock.source.menu_name` 仍会被模板读取；在「菜单」设置页选择新值后，以 `navigation.header.menu_name` / `navigation.dock.menu_name` 为准。模板也兼容过渡期的 `navigation.source.*`。

建议：

- Header 使用结构完整的主导航菜单。
- Dock 单独准备扁平快捷入口菜单。
- 如果一个菜单项既有链接又有子菜单，Header 会优先把它当作分组标题。
- 菜单项元数据中的图标主要服务 Dock，也会用于 Header 二级菜单；顶层 Header 文本菜单不显示图标。
