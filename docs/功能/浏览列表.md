# 浏览列表

## 结论

浏览列表是一个聚合 feature，不是单页 feature。
它统一承接：

- 归档
- 标签页
- 标签详情
- 分类页
- 分类详情
- 作者页

当前目录名已经是：

- [/src/features/browser-explorer](/src/features/browser-explorer)

但协议名暂时还是旧值：

- `pageApp = explorer`
- `pageMode = browser-list`
- `windowVariant = browser`

## 当前统一点

这组页面统一使用：

- browser 风格主窗口
- `browser-list` 页面模式
- explorer 侧边栏 / 列表交互
- 同一套 PJAX 列表切换逻辑

## 什么时候改这里

这些需求优先改 `browser-explorer`：

1. 归档页布局
2. 标签页布局
3. 分类页布局
4. 作者页列表布局
5. explorer 侧边栏交互

如果只是 reader 正文，不要改这里。

## 我想配什么，先配哪里

浏览列表主要靠内容数据和 Halo 本身配置驱动，主题层通常不通过后台单独配置。
如果你只是想调整：

1. 分类图标与颜色：看 [/annotation-setting.yaml](/annotation-setting.yaml) 中 `Category`
2. 标签图标：看 [/annotation-setting.yaml](/annotation-setting.yaml) 中 `Tag`
