# 浏览列表

## 结论

浏览列表是一组聚合 App，不是单页 App。
它统一承接：

- 归档
- 标签页
- 标签详情
- 分类页
- 分类详情
- 作者页

当前源码按页面拆分为：

- [/src/apps/explorer/archives](/src/apps/explorer/archives)
- [/src/apps/explorer/tags](/src/apps/explorer/tags)
- [/src/apps/explorer/categories](/src/apps/explorer/categories)
- [/src/apps/explorer/author](/src/apps/explorer/author)

协议名分别是：

- `pageApp = explorer-archives`
- `pageApp = explorer-tags`
- `pageApp = explorer-categories`
- `pageApp = explorer-author`
- `pageMode = browser-list`
- `windowVariant = browser`

## 当前统一点

这组页面统一使用：

- browser 风格主窗口
- `browser-list` 页面模式
- explorer 侧边栏 / 列表交互
- 同一套 PJAX 列表切换逻辑

## 什么时候改这里

这些需求优先改对应的 `src/apps/explorer/*` 目录：

1. 归档页布局
2. 标签页布局
3. 分类页布局
4. 作者页列表布局
5. explorer 侧边栏交互

如果只是 reader 正文，不要改这里。

## 我想配什么，先配哪里

浏览列表主要靠内容数据和 Halo 本身配置驱动，主题层通常不通过后台单独配置。
如果你只是想调整：

1. 分类图标与颜色：看 [/annotation-setting.yaml](/annotation-setting.yaml) 中 `Category`
2. 标签图标：看 [/annotation-setting.yaml](/annotation-setting.yaml) 中 `Tag`
