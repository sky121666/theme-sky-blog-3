# 链接页

## 结论

链接页已经是独立 app，不再复用普通 browser/explorer 窗口。

当前协议固定为：

- `pageApp = links`
- `pageMode = browser-links`
- `windowVariant = links`

它承接 6 类能力：

1. 友链分组浏览
2. PluginLinks RSS 朋友圈
3. 登录权限用户的收藏与稍后阅读
4. RSS 已读 / 未读管理与内部摘要详情
5. 留言板查看
6. 友链申请 / 管理员直连添加

## 页面结构

links 页使用独立的微信桌面端式三段结构，不再使用 Finder 卡片页：

1. `60px` 功能栏
   - 链接
   - 朋友圈
   - 收藏（仅 `plugin:links:view` 权限用户）
   - 稍后阅读（仅 `plugin:links:view` 权限用户）
   - 添加友链
   - 留言板
2. `440px` 会话式列表
   - 默认直接显示全部友链
   - 朋友圈视图显示“全部动态”和 RSS 来源
   - 搜索和分组筛选只更新列表，不打开新页面
3. 按需展开的详情区
   - 友链资料与访问入口
   - 单一 RSS 来源的头像、简介、域名、访问入口
   - RSS 动态、刷新和游标续载
   - RSS 内部摘要详情、收藏、稍后阅读和已读 / 未读操作
   - 内嵌添加助手
   - 留言板

`/links` 默认只有功能栏与列表，窗口宽度为 `500px`；选中链接、RSS 来源、添加助手或留言板后才平滑展开详情区。左侧功能栏和列表在桌面端始终保留，不会因为进入详情而重建。

配色由 links app 自己固定，只读取明暗模式，不读取主题强调色或 macOS 强调色：

- 主绿色：`#07C160`
- 浅绿色：`#95EC69`
- 淡绿色选中底：`#DFF7E8`

实现继续使用项目现有 Tailwind、Iconify、Alpine 和原生 HTML 控件，没有引入额外 UI 框架。

## 路由状态

| 状态 | 路由 |
| --- | --- |
| 链接列表 | `/links` |
| 分组筛选 | `/links?group={group}` |
| 链接详情 | `/links?link={link}` |
| 朋友圈来源列表 | `/links?view=friends` |
| 全部 RSS 动态 | `/links?view=friends&scope=all` |
| 未读 RSS 动态 | `/links?view=friends&scope=unread` |
| 收藏 | `/links?view=friends&scope=favorite` |
| 稍后阅读 | `/links?view=friends&scope=later` |
| 单一 RSS 来源 | `/links?view=friends&linkName={link}` |
| RSS 内部详情 | 在对应 RSS 路由追加 `&itemId={item}` |
| 添加友链 | `/links?view=apply` |
| 留言板 | `/links?view=board` |

这些状态使用 `history.pushState`，刷新、分享、前进和后退都应恢复同一界面。

## 后台配置与注解

links 页当前涉及两条配置链：

### 1. 主题设置

配置位置：

- [/settings.yaml](/settings.yaml)

字段：

- `links.profile.display_name`

作用：

- 配置“全部友链”默认标题
- 同时影响页面主标题、侧边栏标题和窗口标题回退值

### 2. LinkGroup 注解

配置位置：

- [/annotation-setting.yaml](/annotation-setting.yaml)

对象：

- `LinkGroup`

字段：

- `icon`

作用：

- 友链分组侧边栏图标
- 分组标题图标

留空时回退到主题内置默认图标。

## 提交与留言流程

当前完全按 `PluginLinks 2.2.1` 官方能力分成两条安全路径，不依赖 `plugin-friends` 或 `link-submit`。

### 1. 已登录管理员

- 先读取 Halo 当前用户，再读取该用户的 `uiPermissions`；只有 `plugin:links:manage` 才允许直接新增，不能再用“可查看列表”冒充管理权限。
- 有权限时使用官方 detail 与 RSS discovery 接口识别站点信息。
- 新增友链通过 Halo 标准 `core.halo.run/v1alpha1/links` 资源接口写入，RSS 地址映射到 `spec.rss`，写入请求携带 Halo 当前会话的 CSRF 令牌。
- 修改申请不直接覆盖现有数据，仍生成可审核的留言内容。
- 页面不提供手动提交模式开关：权限成立时新增友链自动直连；访客、无权限用户和修改申请自动进入留言流程。
- 权限失效或接口失败时说明具体原因，并降级到留言申请。

### 2. 访客申请

- 官方插件没有匿名创建接口，因此访客不会请求任何受保护的后台写入端点。
- 浏览器以 `credentials: omit` 直接读取目标站公开 HTML；受目标站 CORS、HTTPS 混合内容和内容类型规则限制。
- 识别成功只填充表单，不额外弹出成功提示；失败会说明跨域、非 HTML、超时或后台权限原因，并保留网址供手动填写。
- 完成后生成 Markdown、复制并切换到 PluginLinks 留言板，由管理员审核处理。

## 朋友圈与 RSS

- 统一入口为 `/links?view=friends`，不再维护 `/friends` 页面和独立 Friends app。
- 来源列表使用 `linkFeedFinder.groupBy(1)`；只有打开“全部动态”、RSS 分组或单一来源详情时才调用 `linkFeedFinder.list({limit: 20})`，默认来源列表不提前读取动态正文。
- 访客和无 RSS 权限用户只请求匿名公共 `/apis/api.link.halo.run/v1alpha1/linkfeeds`，不携带 Halo 登录态。
- 具有 `plugin:links:view` 的登录用户自动改用受保护的 `/apis/console.api.link.halo.run/v1alpha1/rss/items`，并显示独立收藏、稍后阅读和未读入口；公开视图与权限视图共用同一套界面，不提供人工切换。
- 受保护列表按 `read=false`、`favorite=true`、`readLater=true` 映射到未读、收藏和稍后阅读；游标仍使用 `beforePublishedAt + beforeId`。
- 收藏、稍后阅读、已读 / 未读通过 PluginLinks 官方状态接口更新；这些状态由当前 Halo 站点保存，不是浏览器本地状态。
- 权限用户点击动态标题进入内部摘要详情并自动标记已读；从稍后阅读打开时同时完成该任务，收藏状态保持不变，也可以手动重新设为未读。
- 支持 `groupName`、`linkName` 和 `beforePublishedAt + beforeId`；分组与单来源参数不会同时发送。
- RSS 公共能力关闭时显示明确空态；切换、刷新和续载不使用骨架屏，更新期间保留现有列表并轻量淡化。
- `PluginLinks` 契约和严格 inventory 均固定到官方稳定版 `2.2.1`。

相关接口写在：

- [/src/apps/links/runtime.js](/src/apps/links/runtime.js)

验证入口为 `pnpm run verify:links`；真页交互同时纳入 `pnpm run smoke:playwright`。当前真页已确认权限入口、真实收藏列表和只读路由；状态写入由契约测试覆盖，避免自动验证改动管理员现有收藏数据。

## 移动端规则

links 页移动端使用“列表 → 全屏详情”导航：

- 隐藏 menubar
- 隐藏 dock
- 隐藏 traffic lights
- 隐藏 resize handle
- 窗口铺满 `100dvh`
- 功能栏移到底部
- 默认显示单列列表
- 选中链接、RSS 来源、添加助手或留言板后显示全屏详情
- 详情顶部返回按钮回到原列表

如果你看到手机端又出现 menubar / dock，优先检查：

- [/src/apps/links/styles/index.css](/src/apps/links/styles/index.css)

## 什么时候改这里

这些需求优先改 links app：

1. 友链会话式列表与资料详情
2. RSS 朋友圈、来源筛选与继续加载
3. links 页面搜索 / 分组筛选
4. 内嵌添加助手与权限降级
5. 留言板兜底逻辑
6. links 移动端窗口表现

不要把 links 逻辑塞回通用 browser/explorer。
