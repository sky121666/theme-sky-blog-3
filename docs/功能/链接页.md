# 链接页

## 结论

链接页已经是独立 app，不再复用普通 browser/explorer 窗口。

当前协议固定为：

- `pageApp = links`
- `pageMode = browser-links`
- `windowVariant = links`

它承接 3 类能力：

1. 友链分组浏览
2. 留言板查看
3. 友链申请 / 自助提交

## 页面结构

links 页现在固定分成两块：

1. 左侧侧边栏
   - 全部友链
   - 友链分组
   - 添加友链
   - 留言板
2. 右侧主内容区
   - 友链卡片列表
   - 搜索 / 排序
   - 留言板
   - 提交弹窗

这套结构已经包含在 links 自己的窗口里，不应该再回退成“复用别的 browser 窗口内容”。

## 后台配置与注解

links 页当前涉及两条配置链：

### 1. 主题设置

配置位置：

- [/settings.yaml](/settings.yaml)

字段：

- `links.profile.display_name`

作用：

- 配置“全部友链”默认标题
- 同时影响页面主标题、侧边栏标题和窗口标题回退值

### 2. LinkGroup 注解

配置位置：

- [/annotation-setting.yaml](/annotation-setting.yaml)

对象：

- `LinkGroup`

字段：

- `icon`

作用：

- 友链分组侧边栏图标
- 分组标题图标

留空时回退到主题内置默认图标。

## 提交与留言流程

当前有两条路径：

### 1. 自助提交流程

当站点安装了 `link-submit` 插件时：

- 自动加载分组列表
- 支持根据站点地址调用 link detail 接口提取信息
- 支持提交 `add / update`
- 支持选择目标分组

相关接口写在：

- [/src/apps/links/runtime.js](/src/apps/links/runtime.js)

### 2. 留言兜底流程

以下情况会自动回退到留言模式：

1. 没安装 `link-submit` 插件
2. 分组接口加载失败
3. 提交接口返回失败

回退后会：

- 生成 Markdown 申请内容
- 显示“复制并前往留言板”
- 自动切到留言板继续提交

这部分是 links 页的真实兜底能力，不是提示文案。

## 移动端规则

links 页移动端已经做成沉浸式窗口：

- 隐藏 menubar
- 隐藏 dock
- 隐藏 traffic lights
- 隐藏 resize handle
- 窗口铺满 `100dvh`
- 主内容区可滚动

如果你看到手机端又出现 menubar / dock，优先检查：

- [/src/apps/links/styles/index.css](/src/apps/links/styles/index.css)

## 什么时候改这里

这些需求优先改 links app：

1. 友链分组卡片布局
2. links 页面搜索 / 排序
3. 添加友链弹窗
4. 自助提交插件联动
5. 留言板兜底逻辑
6. links 移动端窗口表现

不要把 links 逻辑塞回通用 browser/explorer。
