# 架构总览

## 结论

`theme-sky-blog-3` 是一个 **桌面壳层 + 单主窗口内容区** 的 Halo 主题。
它不是 Finder-first 主题，也不是多页模板堆叠主题。

当前结构的核心原则只有 4 条：

1. 页面一律走公共壳层
2. 页面内容只在内容容器里替换
3. shell 能力归 `desktop-shell`
4. 独立功能归各自 feature 目录

## 页面协议

| 页面族 | `pageApp` | `pageMode` | `windowVariant` | 入口模板 |
| --- | --- | --- | --- | --- |
| 图库 | `photos-app` | `browser-list` | `photos` | [/templates/photos.html](/templates/photos.html) |
| 瞬间列表 / 详情 | `moments-app` | `browser-moments` | `moments` | [/templates/moments.html](/templates/moments.html), [/templates/moment.html](/templates/moment.html) |
| 归档 / 标签 / 分类 / 作者 | `explorer` | `browser-list` | `browser` | [/templates/archives.html](/templates/archives.html) 等 |
| 文章 / 单页 | `reader` | `browser-reader` | `browser` | [/templates/post.html](/templates/post.html), [/templates/page.html](/templates/page.html) |

说明：

- 现在目录名已经用了 `browser-explorer` 和 `browser-reader`
- 但 `pageApp` 协议名还没有统一改名
- 这是当前仓库刻意保留的过渡状态，不是遗漏

## 关键边界

### 1. Shell 与内容的边界

PJAX 只允许替换内容容器，不允许替换：

- 桌面背景
- Header
- Dock
- 主窗口外壳

### 2. 样式边界

- `desktop-shell` 负责固定壳层
- feature 目录负责页面内容样式
- 构建后仍统一输出到 `templates/assets`

### 3. 构建边界

输出根固定为：

- [/templates/assets](/templates/assets)

其中：

- `css/js` 是构建托管目录
- `images/favicon` 是静态资源目录

当前治理入口：

- [/vite.config.ts](/vite.config.ts)

## Token 归属

### 后台注入

由 [/templates/modules/shell/layout.html](/templates/modules/shell/layout.html) 注入基础变量：

- `--mac-accent`
- `--mac-selection`
- `--mac-folder1/2/3`
- `--mac-shell-light`
- `--mac-shell-dark`

### 前端桥接

由 [/src/shell/desktop-shell/styles/desktop/surface.css](/src/shell/desktop-shell/styles/desktop/surface.css) 建立桥接变量：

- `--theme-accent`
- `--theme-selection`
- `--theme-folder1/2/3`
- `--theme-shell-light`
- `--theme-shell-dark`
- `--content-chrome-*`

原则：

- 后台只负责主题值来源
- 前端只通过桥接 token 消费
- 不让组件直接依赖后台原始变量名

## 当前稳定规则

1. `README.md` 只做入口和进度，不做功能详解
2. shell 固定能力归 `desktop-shell`
3. 独立页面能力归 feature
4. 顶层 `templates/*.html` 继续保留为 Halo 路由入口
5. 构建输出根继续固定为 [/templates/assets](/templates/assets)
