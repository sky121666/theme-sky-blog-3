# 发布说明

## v0.9.43

发布时间：2026-07-22

### PluginLinks 2.2.1 重构

- 友链、RSS 朋友圈、收藏、稍后阅读、已读状态、留言板和添加友链助手统一到 `PluginLinks 2.2.1` 官方契约；不再维护独立 Friends 页面或旧自助提交插件通道。
- `/links` 重构为微信桌面端式应用：固定功能栏、会话列表和按需详情区；使用独立微信绿色体系，只跟随明暗模式，并补齐移动端列表/详情切换。
- 访客添加友链只做无凭据浏览器识别，失败时保留手动填写并转到留言板；具 `plugin:links:manage` 权限的登录用户才调用官方 detail/discovery 与 Link CRUD。
- 具 `plugin:links:view` 权限的登录用户可使用未读、收藏、稍后阅读、自动已读和手动设为未读；访客继续使用匿名公开 RSS，不携带站点凭据。
- 桌面朋友圈小组件正式改为 `plugin-links.feed`，数据源为 `linkFeedFinder`；删除 `plugin-friends.recent` 兼容 ID、旧目录和运行时映射。旧布局如曾保存该组件，升级后需要在组件中心重新添加一次。
- 增加反向契约门禁，禁止旧 `friendFinder`、`plugin-friends.recent`、`link-submit` guard、匿名提交 API 和旧 DOM 命名重新进入活动代码。

### Finder、图库与文档体验

- 分类、标签统一为 Finder 三栏结构，根页默认展示全部文档，适配元数据图标/颜色、明确分页路由、轻量选中态和无骨架同 App PJAX。
- 归档改为“年份 → 月份 → 月内文章”层级，上一页/下一页只服务月内文章分页，并支持继续加载、深链和历史恢复。
- 图库保留稳定侧栏，详情新增底部胶片条、键盘/左右切换、平滑方向动画和空闲自动隐藏；修复大图切换或列表进入详情时短暂覆盖胶片条的问题。
- 修复 Docsme 项目卡片悬停合成残影与深色版本选择器配色，补齐 PJAX 首屏样式交接，避免分类、标签、归档切换时短暂显示未加载样式。

### 兼容与发布

- Halo 最低版本保持 `>=2.25.0`，发布前按 Halo `2.25.4` 验证；PluginLinks 契约和已测试版本均为 `2.2.1`。
- 旧 `plugin-friends` 与 `link-submit` 不再属于主题插件清单、路由、Finder、组件或构建契约；文档中的旧版本记录仅作为历史发布事实保留。

- 支持 `PluginPhotos` 2.1.1 的列表页和详情页惰性模型变量。
- 支持 `plugin-friends` 1.4.6 在 Links 2.0.0 依赖下继续使用 SSR feed 和 `linkName` 筛选。
- 支持 `plugin-shiki` 1.3.1 的末尾空代码行修正；主题继续避免重复接管高亮。

### 不支持 / 暂不支持

- 不支持访客或普通用户使用友链自动补全；该功能仅面向管理员。
- 不支持匿名前端调用 Links Console detail API。
- 不支持旧版 `PluginLinks/link-detail` 匿名提取接口。
- 不支持自动判断所有自定义管理员角色；当前识别 `super-role`、`admin`、`administrator`。
- 暂未实测普通登录但非管理员账号；该路径按角色判断隐藏自动补全入口。
- `PluginPhotos` 公共 REST、tag、ungrouped 和详情预取仍属于后续增强，不在本版本接入。
- Docsme / Moments 中 Shiki 真实代码块样本仍待补充，不阻塞当前 `plugin-shiki` 1.3.1 兼容标记。
