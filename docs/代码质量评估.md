# 代码质量评估

## 结论

当前 `theme-sky-blog-3` 的代码质量已经进入 **可持续维护** 状态。

如果把站点内容资源和后端插件包体先排除，当前主题自身的判断是：

1. 架构分层已经清晰
2. 页面主链路已经稳定
3. 工程护栏已经可用
4. 剩余问题主要是内容资源和插件加载边界

- [/src/shell-core](/src/shell-core)

## 当前主要风险

### 1. 内容资源仍会污染真实体验

这类问题不属于主题结构问题，但会在真实页面里体现出来：

- 正文里的外链图片
- about 页内网图片 / 媒体
- 大图资源导致的加载波动

所以“功能验证通过”不等于“所有内容页都完全干净”。

### 2. 插件资源仍然偏重

主题无法控制插件包体，但会受到插件脚本影响：

- `rag-ui.js`
- `comment-widget.js`
- `contact-form.iife.js`

当前主题侧最合理的策略不是“缩插件”，而是：

- 控制插件何时加载
- 控制插件在哪些页面加载

### 3. 共享基础样式仍需严格管理

最典型的例子是 `author` 页会复用 `tag-*` 的基础结构样式。

这类问题现在已经修到可用，但也说明：

- 共享基础样式必须明确来源
- 否则容易再次出现“页面局部样式丢失”

最新审计已增强以下字段：

### 资源审计结论

| 资源 | 当前证据 | 判断 |
| --- | --- | --- |
| `comment-widget` | 8 个采样页全部命中，页面 DOM 未发现评论入口 | 疑似插件经 `<halo:footer />` 全局注入；不在主题侧强拦 |
| `rag-ui` | 8 个采样页全部命中，共 16 次请求，约 5 KB | 体积小但边界不清；先记录，不做拦截 |
| `contact-form` | 8 个采样页全部命中，采样页均无联系表单 | 疑似全局注入；除非插件支持页面级开关，否则不改主题 |
| `shiki` | 8 个采样页全部命中，采样页无代码块 | 疑似全局注入；继续用 Docsme/Moments 样本脚本复验真实代码块 |
| `large-media` | 主要集中在 `/photos`、`/moments`、`/equipments`，图片 lazy 比例正常 | 属于真实内容资源；没有性能异常前不做虚拟列表或全局缓存 |

### 当前性能结论

如果排除内容图和插件资源：

- `shell-core` 已稳定
- `apps` 拆分没有引入明显主线程问题
- 图片懒加载已经在 `moments / author / reader / photos / douban` 等主题可控图片场景生效

## 当前建议

### 优先级 1

- 继续保持 `route-manifest` 作为单一来源
- 新页面必须进入 `src/apps/*`
- 不允许旧 `features/*` 回流

### 优先级 2

- 把图片懒加载继续当成主题默认能力维护
- 避免新模板再写回同步首屏大图

### 优先级 3

- 对插件资源做页面级加载控制
- 不让所有插件资源默认挤进首屏
- 使用 [/docs/验证与审计矩阵.md](/docs/验证与审计矩阵.md) 先确认误加载或真实性能问题，再做最小修复
