# 小组件扩展设计

## 结论

下一批小组件先做三类：

1. 追番小组件
2. 豆瓣收藏小组件
3. Steam 小组件
4. Equipment 随机装备小组件

`Douban` 独立 App 已完成并确认 `plugin-douban` `1.2.5` 契约；豆瓣收藏小组件进入桌面组件中心，但只做大卡片展示，不扩展成完整筛选面板。

本轮目标不是把插件页面搬进桌面，而是做轻量、稳定、可配置的桌面入口。小组件只消费已经确认的 Finder 或模板注入数据，不解析页面 DOM，不新增复杂后台设置。

## 评分目标

这份设计按 **10/10 可开工标准** 收口：

| 维度 | 标准 |
| --- | --- |
| 数据契约 | 每个小组件都有明确 Finder、字段、缺失降级和禁止推断规则 |
| 交互契约 | 预览不跳转、主卡跳转清晰、无复杂二级操作 |
| 配置契约 | 每个 `meta` 字段都有类型、默认值、合法值和保存回读要求 |
| 视觉契约 | 每个小组件有明确视觉基调、图像策略和明暗模式要求 |
| 验证契约 | 每批都有专属验证脚本或验收断言，不只依赖通用 smoke |
| 实施契约 | 文件清单、提交边界、停止条件都能直接执行 |

## 通用设计原则

### 数据边界

- 插件小组件只在对应插件可用时出现在组件中心。
- 数据优先在 [/templates/modules/shell/layout.html](/templates/modules/shell/layout.html) 中通过 Finder 判空后注入。
- 前端渲染器优先消费 `sources`；豆瓣小组件例外，它消费 `plugin-douban` 明确暴露的公开 API，不解析页面 DOM。
- 无数据时显示空态，不伪造统计、状态、评分或在线信息。

### Source 命名约定

新增 sources 统一使用复数数据名和显式可用性标记：

| Source | 说明 |
| --- | --- |
| `bangumisAvailable` | Bilibili 追番插件是否可用 |
| `bangumisByStatus` | 按状态预取的追番/追剧条目集合 |
| `bangumiStatusCounts` | 想看 / 在看 / 已看统计 |
| `steamAvailable` | Steam 插件是否可用 |
| `steamProfile*`、`steamStats*`、`steamRecentGames` | 现有 Steam sources，保持不改名 |
| `doubanAvailable` | Douban 插件是否可用 |
| `doubanUrl`、`doubanApiBase` | 豆瓣归档页与公开 API 入口 |
| `equipmentsAvailable` | Equipment 插件是否可用 |
| `equipmentItems` | 预取装备条目 |
| `equipmentGroups` | 装备分组 |

### 交互边界

- 点击主卡进入对应独立 App：
  - 追番：`/bangumis`
  - 豆瓣：`/douban`
  - Steam：`/steam`
  - Equipment：`/equipments`
- 组件中心预览模式必须禁用 PJAX 跳转。
- 小组件内部不提供复杂二级筛选；筛选和完整浏览交给独立 App。

### 配置边界

- 只使用实例级 `meta`，写入 `default_layout.layout_json.instances[].meta`。
- 不新增全局后台设置，避免把桌面小组件变成另一套插件配置面板。
- 配置项必须通过 manifest 声明 `hasConfig`、`configSchema`、`configDefaults`。

需要扩展配置系统时，只扩展通用控件，不在单个 widget 里写专用弹窗。

| 控件类型 | 用途 | 第一批使用方 |
| --- | --- | --- |
| `select` | 固定选项、状态、模式 | 追番、豆瓣、Steam、Equipment |
| `toggle` | 二元开关 | Steam |
| `equipment-group` | 装备分组选择 | Equipment |

`equipment-group` 的交互形态复用现有 `photo-group` 的列表按钮模式，但数据来自 `sources.equipmentGroups`。

### 样式边界

- 每个新增小组件使用独立 CSS 文件，放在：
  - [/src/shell/desktop-shell/styles/widgets/content](/src/shell/desktop-shell/styles/widgets/content)
- 颜色通过 `--widget-card-*`、`--theme-accent` 或组件局部 token 收口。
- `follow`、`light`、`dark` 三种外观都要可读。

### 视觉质量门槛

| 项目 | 要求 |
| --- | --- |
| 文本 | 正常文本对比度不低于 4.5:1，弱化文本不低于 3:1 |
| 图片 | 图片加载前后不改变卡片尺寸 |
| 动效 | `prefers-reduced-motion: reduce` 下禁用位移和缩放动画 |
| 空态 | 空态要占满卡片主视觉，不显示半截布局 |
| 暗色 | 不使用大面积单一蓝紫渐变；优先中性暗底 + 小面积插件特征色 |
| 移动端 | 遵循现有 `hide_on_mobile`，不单独绕开桌面小组件开关 |

## 批次一：追番小组件

状态：已实现。

### 目标

展示追番/追剧状态入口，优先服务“我最近在看什么”。

### Widget ID

`plugin-bangumis.recent`

### 数据来源

建议在 shell 模板注入：

- `bangumisAvailable`
- `bangumisByStatus`
- `bangumiStatusCounts`

可复用当前 `/bangumis` 页面已经验证过的插件契约：

- `bangumiFinder.list(...)`
- 类型：追番 / 追剧
- 状态：想看 / 在看 / 已看

### 追番契约表

现有 `/bangumis` 页面已使用如下值：

| 字段 | 值 | 含义 |
| --- | --- | --- |
| `typeNum` | `1` | 追番 |
| `typeNum` | `2` | 追剧 |
| `status` | `0` | 全部 |
| `status` | `1` | 想看 |
| `status` | `2` | 在看 |
| `status` | `3` | 已看 |

shell 预取建议：

| Source | Finder 调用 | 用途 |
| --- | --- | --- |
| `bangumisByStatus.wish` | `bangumiFinder.list(typeNum, 1, 1, 4)` | 想看候选 |
| `bangumisByStatus.watching` | `bangumiFinder.list(typeNum, 2, 1, 4)` | 默认展示 |
| `bangumisByStatus.done` | `bangumiFinder.list(typeNum, 3, 1, 4)` | 已看候选 |
| `bangumiStatusCounts.wish` | `bangumiFinder.list(typeNum, 1, 1, 1).total` | 统计 |
| `bangumiStatusCounts.watching` | `bangumiFinder.list(typeNum, 2, 1, 1).total` | 统计 |
| `bangumiStatusCounts.done` | `bangumiFinder.list(typeNum, 3, 1, 1).total` | 统计 |

v1 不做“全部”大列表预取。`status = auto` 时按 `watching -> wish -> done` 回退。

可用字段限定：

| 字段 | 用途 |
| --- | --- |
| `metadata.name` | 稳定 key，缺失时用标题兜底 |
| `spec.title` | 标题 |
| `spec.cover` | 封面 |
| `spec.url` | 外链或详情入口 |
| `spec.score` | 评分，可选 |
| `spec.totalCount` | 集数，可选 |
| `spec.type`、`spec.area` | 标签，可选 |
| `spec.des` | 摘要，仅 medium/large 使用 |

### 尺寸设计

| 尺寸 | 内容 |
| --- | --- |
| `small` | 一张当前在看卡片，封面铺底，显示名称和状态 |
| `medium` | 左侧状态计数，右侧展示 1-2 个条目 |
| `large` | 顶部状态分段，下面展示 4 个条目，底部进入 `/bangumis` |

### 配置项

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `typeNum` | `select` | 追番 / 追剧 / 全部 |
| `status` | `select` | 在看 / 想看 / 已看 / 自动 |

默认：

```json
{
  "typeNum": "",
  "status": "auto"
}
```

合法值：

| 字段 | 合法值 |
| --- | --- |
| `typeNum` | `""`、`"1"`、`"2"` |
| `status` | `"auto"`、`"watching"`、`"wish"`、`"done"` |

实现时要把 `status` 映射到插件数值：`wish -> 1`、`watching -> 2`、`done -> 3`。

### 空态

- 插件未安装：不出现在组件中心。
- 插件安装但无条目：显示“还没有追番记录”，点击进入 `/bangumis`。
- 封面缺失：使用图标占位，不请求外部随机图。

### 视觉规格

| 项目 | 规格 |
| --- | --- |
| 基调 | 封面驱动的媒体卡，像“正在看的内容”而不是数据面板 |
| 主色 | 使用封面叠层 + `--theme-accent` 小面积状态色 |
| 图像 | 封面优先，缺失用 `lucide-tv-minimal` 或 `lucide-sparkles` |
| small | 方形封面卡，标题最多 2 行 |
| medium | 左侧统计，右侧 1-2 个横向条目 |
| large | 顶部状态分段，4 个条目固定网格 |

### 专属验证脚本

新增：

- `scripts/verify-widget-bangumis.mjs`

断言：

- `status = auto` 时按 `watching -> wish -> done` 回退。
- `typeNum = "1"` 只取追番 source，`typeNum = "2"` 只取追剧 source。
- 封面缺失时输出占位，不输出空 `img src=""`。
- 预览模式下主卡链接不带可触发 PJAX 的行为。

### 实现文件

## 批次二：Steam 小组件

状态：已按中卡片落地，只支持 `medium`。

### 目标

提供 Steam 状态入口，不复制完整 Steam 页面。

### Widget ID

`plugin-steam.summary`

### 数据来源

当前 shell 已经注入 Steam 数据：

- `steamAvailable`
- `steamPlaying`
- `steamStatusText`
- `steamPersonaName`
- `steamAvatarFull`
- `steamProfileUrl`
- `steamLevel`
- `steamStatsTotalGames`
- `steamStatsRecentPlaytimeFormatted`
- `steamRecentGames`

### Steam 字段契约

| Source | 字段 | 允许用途 |
| --- | --- | --- |
| `steamPlaying` | boolean | 唯一“正在游戏”判断来源 |
| `steamStatusText` | string | 展示在线/游戏状态文案 |
| `steamPersonaName` | string | 昵称 |
| `steamAvatarFull` | string | 头像 |
| `steamProfileUrl` | string | 外部资料页，可选 |
| `steamLevel` | number | 等级 |
| `steamStatsTotalGames` | number | 游戏总数 |
| `steamStatsRecentPlaytimeFormatted` | string | 最近游玩时长 |
| `steamRecentGames` | array/page | 最近游戏，只能显示“最近玩过” |

### 状态规则

优先级：

1. `steamPlaying === true`：显示正在游戏。
2. `steamPlaying !== true` 且有 `steamRecentGames`：只能显示“最近玩过”。
3. 有资料和统计：显示资料摘要。
4. 无数据：显示 Steam 入口空态。

禁止：

- 不从最近游玩推断正在游戏。
- 不轮询、不 websocket、不做实时在线。
- 不把 Steam 页面大面积视觉复制到小组件。
- 不把 `steamProfileUrl` 作为站内 PJAX 链接；外链必须 `target="_blank"`。

### 尺寸设计

| 尺寸 | 内容 |
| --- | --- |
| `medium` | 头像 + 状态 + 当前游戏/最近游戏 + 统计 |

不实现 `small` / `large`。后续如果需要扩展尺寸，必须重新确认信息密度和拖拽稳定性。

### 配置项

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `showRecentGame` | `toggle` | 是否显示最近游戏 |
| `showStats` | `toggle` | 是否显示统计 |

默认：

```json
{
  "showRecentGame": true,
  "showStats": true
}
```

### 视觉规格

| 项目 | 规格 |
| --- | --- |
| 基调 | 安静的状态卡，不是 Steam 客户端缩略图 |
| 主色 | 中性深灰/浅灰底，在线状态可用 Steam cyan，正在游戏可用绿色 |
| 图像 | 头像是主视觉；只有 `steamPlaying === true` 时才允许使用当前/最近游戏图作为背景 |
| medium | 头像、昵称、状态、当前游戏或最近游戏、统计 |

### 专属验证脚本

新增：

- `scripts/verify-widget-steam.mjs`

断言：

- `steamPlaying: false` 且 `steamRecentGames` 有数据时，不能输出“正在游戏”。
- `steamPlaying: true` 时可以输出“正在游戏”或 `steamStatusText`。
- `showRecentGame: false` 时不输出最近游戏标题。
- `showStats: false` 时不输出总游戏数和最近时长。
- 外链资料页不走 PJAX。

### 实现文件

- 新增：`src/widgets/plugin/steam-summary/manifest.js`
- 新增：`src/widgets/plugin/steam-summary/render.js`
- 修改：`src/widgets/registry.js`
- 修改：`src/widgets/loaders.js`
- 修改：`src/widgets/catalog.js`
- 新增：`src/shell/desktop-shell/styles/widgets/content/steam.css`
- 修改：`src/shell/desktop-shell/styles/widgets/content/index.css`

### 验证

- 正在游戏状态只由 `steamPlaying === true` 触发。
- 最近游玩只显示为“最近玩过”，不能显示成“正在游戏”。
- 插件未安装时组件中心不出现。
- `medium` 尺寸不溢出，悬停和点击不触发布局位移。

## 批次三：Equipment 随机装备小组件

### 目标

提供一个“今日装备 / 随机装备”入口，让装备页在桌面有轻量展示。

### Widget ID

`plugin-equipment.random`

### 数据来源

建议在 shell 模板注入：

- `equipmentsAvailable`
- `equipmentItems`
- `equipmentGroups`

优先复用 `/equipments` 页面已经验证过的 Finder 数据，不接入新公开 API，除非后续要做 headless 外部组件。

### Equipment 契约表

当前主题页面已经使用：

| Source | Finder 调用 | 用途 |
| --- | --- | --- |
| `equipmentItems` | `equipmentFinder.list(1, 8)` | 小组件候选装备 |
| `equipmentGroups` | `equipmentGroupFinder.listAll()` 或插件现有分组 Finder | 分组配置和显示 |
| `equipmentTotal` | `equipmentFinder.list(1, 1).total` | 空态/统计 |

实现前必须先确认分组 Finder 的实际名称。若本地插件未暴露稳定分组 Finder，本批只做不带分组配置的 v1，不临时解析 `/equipments` DOM。

可用字段以当前模板为准：

| 字段 | 用途 |
| --- | --- |
| `metadata.name` | 稳定 key |
| `spec.displayName` 或 `spec.name` | 名称 |
| `spec.description` | 摘要 |
| `spec.logo`、`spec.cover`、`spec.image` | 图片候选，按实际字段确认 |
| `spec.groupName` | 分组过滤 |
| `status.permalink` 或 `/equipments` | 入口 |

如果字段名称和上表不同，先更新本文档再实现。

### 随机策略

随机必须稳定，避免每次 render 闪动。

建议：

- 默认按日期生成 seed，每天固定一件。
- 如果实例 `meta.groupName` 存在，则只在该分组内随机。
- 如果分组无数据，回退全量装备。

seed 规则：

```text
seed = `${widget.key}:${YYYY-MM-DD}:${groupName || 'all'}`
```

不能使用 `Math.random()` 直接决定渲染结果。

### 尺寸设计

| 尺寸 | 内容 |
| --- | --- |
| `small` | 单个装备图 + 名称 |
| `medium` | 单个装备图 + 名称 + 分组/描述 |
| `large` | 今日装备 + 2-3 个候选装备小条 |

### 配置项

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `groupName` | `select` | 限定装备分组 |
| `mode` | `select` | 今日随机 / 最新收录 |

默认：

```json
{
  "groupName": "",
  "mode": "daily"
}
```

合法值：

| 字段 | 合法值 |
| --- | --- |
| `groupName` | `""` 或已存在分组 `metadata.name` |
| `mode` | `"daily"`、`"latest"` |

### 空态

- 插件未安装：不出现在组件中心。
- 无装备数据：显示“还没有装备记录”，点击进入 `/equipments`。
- 无图片：使用设备图标占位，不拉外部图。

### 视觉规格

| 项目 | 规格 |
| --- | --- |
| 基调 | 物品陈列卡，像桌面上的一件装备 |
| 主色 | 中性玻璃底 + 低饱和金属/灰阶，不做强烈橙棕色主题 |
| 图像 | 图片优先；无图片使用 `lucide-package` 或 `lucide-boxes` |
| small | 单图单标题，适合做桌面图标旁的小陈列 |
| medium | 大图 + 名称 + 分组/摘要 |
| large | 今日装备主卡 + 2-3 个候选条目 |

### 专属验证脚本

新增：

- `scripts/verify-widget-equipment.mjs`

断言：

- 同一 `widget.key + 日期 + groupName` 多次渲染结果一致。
- `mode = latest` 不走随机，取列表第一项。
- `groupName` 无匹配数据时回退全量装备并输出可读空态/候选。
- 无图片时不输出空 `img src=""`。
- 保存回读后 `meta.groupName` 和 `meta.mode` 不丢失。

### 实现文件

- 新增：`src/widgets/plugin/equipment-random/manifest.js`
- 新增：`src/widgets/plugin/equipment-random/render.js`
- 修改：`src/widgets/registry.js`
- 修改：`src/widgets/loaders.js`
- 修改：`src/widgets/catalog.js`
- 修改：`templates/modules/shell/layout.html`
- 修改：`templates/modules/shell/desktop-widgets.html`
- 新增：`src/shell/desktop-shell/styles/widgets/content/equipment.css`
- 修改：`src/shell/desktop-shell/styles/widgets/content/index.css`

### 验证

- 同一天同一实例显示同一件装备。
- 改 `groupName` 后只在对应分组内选择。
- 保存默认布局后 `meta.groupName` 和 `meta.mode` 可回读。
- 无图、无描述、无分组时都有稳定降级。

## 实施顺序

建议按以下顺序做：

1. 追番小组件
2. 豆瓣收藏小组件
3. Steam 小组件
4. Equipment 随机装备小组件

原因：

- 追番数据结构最接近已有页面列表，风险最低。
- 豆瓣已有公开 API 和独立 App，可先用大卡片验证收藏展示体验。
- Steam 数据已经注入 shell，但状态语义必须谨慎，适合作为第二批。
- Equipment 需要设计稳定随机和分组过滤，逻辑稍多，放第三批。

每批独立提交，不混在一个大提交里。

每批提交前必须确认：

- `git status --short` 只包含本批相关文件。
- 构建产物和源码属于同一次 `pnpm run check`。
- 不创建分支，仍在 `main` 上工作。

## 停止条件

任一批出现以下情况，停止继续扩展，先修当前问题：

- `pnpm run verify:reload` 失败。
- Playwright smoke 在组件中心、首页桌面、`/bangumis`、`/douban`、`/steam`、`/equipments` 任一路径失败。
- 插件 Finder 字段和本文档契约不一致。
- 新增小组件导致已有 Photos、Moments、Friends、Docsme 小组件不出现在组件中心。
- `default_layout.layout_json` 保存后丢失 `meta`。

## 门禁

每批完成后至少执行：

```bash
node scripts/verify-widgets-config-schema.mjs
pnpm run check
pnpm run verify:reload
SMOKE_BASE_URL=${HALO_BASE_URL:-http://localhost:8090} pnpm run smoke:playwright
```

批次专属脚本：

```bash
node scripts/verify-widget-bangumis.mjs
node scripts/verify-widget-steam.mjs
node scripts/verify-widget-equipment.mjs
```

只需要执行当前批次对应脚本；全部小组件完成后再执行三者全量。

如果改动只涉及文档，可以只执行：

```bash
git diff --check
```
