# 插件适配契约

## 目的

这里记录主题和 Halo 插件之间的前台契约版本。

“契约版本”不是当前站点安装的插件版本，也不是本地技能记录的最新插件版本，而是主题模板、运行时和样式实际按哪个插件版本的路由、模板变量、Finder、API、DOM 片段和前端生命周期编写。

如果无法证明某个插件当时按哪个版本适配，就标记为 `unconfirmed`，不要猜版本。只有在存在提交时间、技能更新时间、旧文档或明确需求文本互相印证时，才允许标记为 `inferred`，并且必须保留“推断”状态。

## 状态定义

| 状态 | 含义 | 是否允许写入模板版本 |
| --- | --- | --- |
| `confirmed` | 有文档、提交或验证记录能证明契约版本 | 是 |
| `compatible-tested` | 契约版本可能较旧，但某个新版本已经实际测试通过 | 是，同时记录已测试版本 |
| `inferred` | 版本由提交时间、技能更新时间、旧文档或需求文本推断得出，但缺少直接验收记录 | 是，但必须同时写 `status: inferred` |
| `split-contract` | 同一插件不同页面或能力按不同时间/字段适配，需要拆分 surface 记录 | 只在具体 surface 行里写版本 |
| `unconfirmed` | 主题已适配该插件，但无法证明契约版本 | 否 |
| `not-adapted` | 主题明确不适配该插件 | 否 |

## 版本字段

| 字段 | 含义 |
| --- | --- |
| 契约版本 | 主题实现写代码时依赖的插件能力版本 |
| 已测试版本 | 当前主题实际验证通过的插件版本 |
| 最新版状态 | 是否已经跟踪和评估插件最新版本 |
| 适配提交 | 能定位主题适配代码的主要提交 |
| 依据 | 文档、提交、验证报告或明确用户决策 |

## 当前收口结论

当前已纳入主题的插件页面和组件均已记录契约版本、已测试版本、状态、模板注释和依据；没有 `unconfirmed` 的已适配插件。

剩余事项分两类：

1. `plugin-douban` 已确认到 `1.2.5`；后续只在插件升级、封面代理配置策略变化或体验增强时重新评估。
2. `plugin-shiki` 的 Docsme/Moments 真实代码块样本仍是样本依赖型复验，不阻塞当前 1.3.1 兼容标记。

## 模板注释规范

对应插件模板只写 HTML 注释，不在未知版本时写版本属性。`inferred` 可以写入具体版本，但必须保留 `status: inferred`，不能升级成 `confirmed`。

已确认契约：

```html
<!-- plugin-contract: plugin-bilibili-bangumi; contract-version: 1.4.0; tested-version: 1.4.0; status: confirmed; source: docs/插件适配契约.md -->
```

未知契约：

```html
<!-- plugin-contract: PluginPhotos; contract-version: pending; status: unconfirmed; source: docs/插件适配契约.md -->
```

推断契约：

```html
<!-- plugin-contract: PluginLinks; contract-version: 2.0.0; status: inferred; source: docs/插件适配契约.md -->
```

分面契约：

```html
<!-- plugin-contract: PluginPhotos; surface: detail; contract-version: 2.1.1; status: confirmed; source: docs/插件适配契约.md -->
```

如果以后验证了新版本兼容：

```html
<!-- plugin-contract: PluginPhotos; contract-version: pending; tested-version: 2.1.1; status: compatible-tested; source: docs/插件适配契约.md -->
```

## 更新流程

1. 更新插件或适配主题前，先查本文件。
2. 对比插件字段、路由、Finder、API、模板变量和 PJAX 生命周期变化。
3. 修改主题后记录验证命令和验证结果。
4. 更新本文件的契约版本、已测试版本、状态和依据。
5. 只有 `confirmed`、`compatible-tested` 或 `inferred` 才允许在模板注释里写具体版本；`inferred` 必须保留推断状态。
6. 同一插件存在不同 surface 时，不要用一个版本覆盖全部能力，按页面或能力拆行记录。
