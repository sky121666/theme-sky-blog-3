# 发布与回滚

## 不变量

1. `v*` 标签只能指向 `main` 可达提交。
2. 构建、Halo 2.25.0/2.25.4 验证和发布必须消费同一个已校验 ZIP；发布作业再次核对 SHA-256。
3. GitHub Release 在 Halo 应用市场同步成功前始终保持草稿。
4. 已发布版本不可覆盖，也不自动删除公开 Release、标签或应用市场版本；修复必须发布更高补丁版本。
5. 新版本公开前必须确认上一稳定 Release 为非草稿，并至少保留一个可下载的上一稳定 ZIP。

规则依据：

- [Halo 官方发布应用文档](https://docs.halo.run/developer-guide/app-store/publish-app)：已发布版本不应修改对应制品，修复问题应发布新版本。
- [GitHub 官方 Release 管理文档](https://docs.github.com/en/repositories/releasing-projects-on-github/managing-releases-in-a-repository)：草稿、发布和删除属于不同的 Release 状态操作。

## 状态恢复矩阵

| 失败位置 | 外部状态 | 允许的恢复方式 | 禁止操作 |
| --- | --- | --- | --- |
| 构建、审计或 Halo 矩阵失败 | 只有远端标签，无 Release | 修复同一提交前先停止发布；确认标签目标后从 Actions 重新运行失败工作流 | 不得绕过失败门禁手工创建公开 Release |
| GitHub 草稿和 ZIP 已创建，但应用市场步骤尚未执行 | GitHub 草稿和已校验 ZIP；无 Halo 草稿 | 明确确认应用市场从未同步后，运行 `Release Recovery`，输入精确标签与失败 CD run ID；工作流校验同一提交和双份 ZIP SHA-256 后继续既有草稿 | 禁止删除或改写标签；禁止用于应用市场已经执行过的 run |
| GitHub 草稿创建后，应用市场同步失败 | GitHub 草稿；Halo 可能遗留草稿 | 先在 Halo 开发者中心确认并删除本次失败草稿，再删除同标签 GitHub 草稿，最后重跑 CD | 不得保留未知 Halo 草稿后盲目重跑 |
| 应用市场成功、GitHub 仍为草稿 | Halo 版本已发布；GitHub 草稿存在 | 直接公开既有 GitHub 草稿 | 禁止删除草稿、禁止重跑 CD、禁止再次创建 Halo 版本 |
| GitHub 与应用市场均已公开后发现严重回归 | 两端版本均公开 | 站点管理员恢复上一稳定 ZIP；维护者保留问题版本证据，发布更高补丁版本；需要暂停分发时联系 Halo 应用市场处理 | 禁止覆盖原 ZIP、禁止删除标签、禁止用相同版本号重发 |

所有删除动作都必须在执行前再次核对精确版本号、标签和草稿 ID；工作流不自动执行这些删除。

`Release Recovery` 只处理“GitHub 草稿与 ZIP 已完成、Halo 应用市场步骤从未执行”的窄状态。它必须同时确认：来源是失败的 CD tag push、run 的 `head_sha` 等于标签提交、Release 仍是草稿、Actions 制品与草稿 ZIP 的 SHA-256 相同。任一条件不成立都会停止，不会自动删除 Release、标签或应用市场版本。

## 发布前门禁

- `pnpm outdated --format json` 返回 `{}`。
- `pnpm run check`、`pnpm run audit:licenses`、经明确授权后的 `pnpm run audit:security` 全部通过。
- 干净冻结安装、发布包校验、Halo 2.25.0/2.25.4 Playwright、性能和 PJAX 门禁全部通过。
- 仓库 `HALO_PAT` 具有应用市场“版本管理”权限，`NPM_AUDIT_ALLOWED=true` 已获维护者授权。
- `v0.9.40` 等上一稳定 Release 为公开状态且包含上一稳定 ZIP；CD 会再次自动验证并写入 Actions 摘要。

## 发布后核验

1. GitHub Release 为非草稿，标签、标题、说明和 ZIP 版本一致。
2. GitHub Release ZIP 的 SHA-256 与 CD 构建作业记录一致。
3. Halo 应用市场出现相同版本，兼容范围为 `>=2.25.0`。
4. 从公开 Release 下载 ZIP，再运行 `node scripts/verify-release-package.mjs <zip>`。
5. 保留上一稳定 ZIP，不因新版本发布而删除回滚基线。
