# 开发约束

## 结论

新增页面、组件、脚本时，默认遵守下面 8 条：

1. 页面只进 `src/apps/*`
2. 小组件只进 `src/widgets/*`
3. 公共壳层只进 `src/shell-core/*` 或 `src/shell/desktop-shell/*`
4. 协议字段只认单一来源
5. 新功能必须至少跑 `pnpm run check`
6. 不允许把旧 `features/*` 再引回来
7. Git 只使用 `main` 分支，不创建功能分支或临时 worktree
8. 视觉风格先判断层级：默认跟随主题，只有特殊视觉需求才显式声明

## 目录约束

### 页面

新页面必须放到：

- [/src/apps](/src/apps)

页面目录最少包含：

- `entry.js`
- `manifest.js`
- `protocol.js`
- `state.js`
- `hydrate.js`

可选：

- `runtime/`
- `styles/`

### 小组件

新组件必须放到：

- [/src/widgets](/src/widgets)

组件目录最少包含：

- `manifest.js`
- `render.js`

### 公共壳层

只能放到：

- [/src/shell-core](/src/shell-core)
- [/src/shell/desktop-shell](/src/shell/desktop-shell)
- [/templates/modules/shell](/templates/modules/shell)

## import 约束

### 允许

- `apps/* -> shared/*`
- `apps/* -> shell-core 暴露接口`
- `apps/explorer/* -> apps/explorer/shared/*`

### 禁止

- `apps/* -> 其他 app 私有 runtime`
- `apps/* -> 旧 features runtime`
- `widgets/* -> 页面 DOM 私有结构`

## 协议约束

### Route

路由归类只能改：

- [/src/shell-core/runtime/route-manifest.js](/src/shell-core/runtime/route-manifest.js)

禁止：

- 在模板里写 app 判定正则
- 在 PJAX 里再写第二套路由归类

### App manifest

页面行为 contract 只能改：

- [/src/shell-core/runtime/app-manifests.js](/src/shell-core/runtime/app-manifests.js)
- [/src/apps](/src/apps)

禁止：

- 在 `pjax/index.js` 里重新维护 same-variant 白名单

### 页面协议

页面必须输出：

- `data-app-root="appId"`
- `data-app-props="appId"`

禁止：

- 散落业务状态到大量 `data-*`

## 视觉与颜色模式约束

### 基本原则

本主题不是把所有页面统一成同一种暗亮色外观，而是模拟 macOS 桌面模型：

- Shell 层统一：桌面、Dock、菜单栏、窗口框架、标题栏、搜索弹窗和系统控件保持统一 macOS 质感。
- App 层可独立：有后端配置、品牌风格、强制暗色、强制亮色或用户自定义色时，可以保留自己的业务视觉。
- Widget 层可配置：桌面小组件继续使用独立 appearance 协议，不强行跟随 App 或 Shell。

不要把独立 App 视觉风格当成需要消除的技术债。没有特殊配置或特殊视觉诉求的 App 默认跟随主题，不需要额外声明。

### 颜色模式分类

开发新组件前先判断它属于哪一种：

| 模式 | 含义 | 适用场景 |
| --- | --- | --- |
| `theme` | 跟随全局 `light / dark / system` | Shell、普通列表、默认系统面板 |
| `light` | 强制亮色 | 图片预览、封面展示、需要保持明亮观感的内容 |
| `dark` | 强制暗色 | Steam、游戏、控制台、沉浸式面板 |
| `brand` | 独立业务风格 | 豆瓣、微信风瞬间、Bilibili 风追番等 |
| `custom` | 用户自定义色 | 后台显式配置背景、文字、边框、强调色的模块 |

自定义颜色不等于独立风格。`color-mode` 决定是否跟随主题，自定义 token 只决定具体颜色。

### 现有协议

当前已有协议继续保留：

- 全局主题：`html[data-color-scheme="light|dark|system"]`
- Widget：`data-widget-appearance="follow|light|dark"`
- Douban App：`data-douban-color-mode="douban|theme"`
- Moments App：`moments-color-theme` / `moments-color-wechat`

这些协议短期不强制迁移。后续改到对应 App 时，可以补充统一说明属性，但不要为了统一而破坏现有视觉。

### 新增组件规则

新增 App 或大型组件时，默认跟随主题，不需要额外声明视觉协议。只有存在后端配置、品牌风格、强制暗色、强制亮色或用户自定义色时，才使用组件已有配置或局部 token 表达，并在对应功能文档中说明原因。

### 样式约束

允许：

- App 使用自己的 token 前缀，例如 `--douban-*`、`--wc-*`、`--steam-*`
- App 保持独立品牌色、强制暗色或强制亮色
- Widget 使用 `appearanceSupport` 明确支持 `follow / light / dark`
- 默认跟随主题的 App 不写额外视觉属性

禁止：

- 用全局 `.dark .xxx` 误伤本应固定品牌风格的 App
- 为每个新组件随意发明一套颜色模式命名但不写文档
- 因为 Shell 统一就强制 App 也统一视觉
- 在主题切换动画里使用全局 `transition: all`

主题切换过渡只应该影响跟随主题的系统层和默认组件。独立 App 风格、强制亮色、强制暗色和品牌风格默认不要求参与全局变色。

## 标题栏与窗口约束

### 标题栏

只能通过这些片段扩展：

- [/templates/modules/shell/window-titlebar.html](/templates/modules/shell/window-titlebar.html)
- [/templates/modules/shell/window-titlebar-leading.html](/templates/modules/shell/window-titlebar-leading.html)
- [/templates/modules/shell/window-titlebar-actions.html](/templates/modules/shell/window-titlebar-actions.html)

禁止：

- 在页面模板里重新写一整套标题栏骨架

### 内容区

窗口内容容器只能走：

- [/templates/modules/shell/window-frame.html](/templates/modules/shell/window-frame.html)

禁止：

- 页面自己重新定义 `data-window-content-root`
- PJAX 替换窗口外壳

## 校验命令

开发时至少执行：

```bash
pnpm run check
```

可单独执行：

```bash
pnpm run lint
pnpm run typecheck
pnpm run smoke
pnpm run build-only
pnpm run smoke:playwright
```

浏览器级 smoke 需要提供站点地址：

```bash
SMOKE_BASE_URL="https://your-site.example.com" pnpm run smoke:playwright
```

可选详情页路径：

```bash
SMOKE_READER_PATH="/posts/demo"
SMOKE_AUTHOR_PATH="/author/demo"
SMOKE_MOMENT_DETAIL_PATH="/moments/demo"
```

## CI 约束

CI 在：

- [/.github/workflows/ci.yaml](/.github/workflows/ci.yaml)

当前默认执行：

- `pnpm run check`
- 完整依赖许可证审计与第三方声明一致性检查
- 经 `NPM_AUDIT_ALLOWED=true` 明确授权后的 npm 漏洞审计；变量缺失时工作流失败且不发送依赖树
- 版本、敏感路径、重复条目与 SHA-256 发布包校验
- 上一稳定 Release/ZIP 回滚基线、GitHub 草稿顺序和失败恢复静态契约
- 固定镜像摘要的 Halo `2.25.0` / `2.25.4` 临时容器矩阵
- 每个 Halo 版本自动初始化、安装/启用/重载主题，再执行 Playwright、HTML 预算和 20 轮 PJAX 生命周期门禁

CI 不再依赖可选的外部 `SMOKE_BASE_URL`，也不会因为变量缺失而静默跳过浏览器门禁。

## 不要再做的事

1. 不要把页面 runtime 再放回 `src/features/*`
2. 不要在模板中复制整套标题栏或窗口 body
3. 不要在 widget render 里直接依赖页面私有 DOM
4. 不要绕过 `asset-manifest.json` 手写资源索引
5. 页面入口产物默认走 `assets/js/apps/<dir>/index.js` 与 `assets/css/apps/<dir>/index.css`

说明：

- `appId` 不一定等于落盘目录名
- 当前 explorer 映射为：
  - `explorer-tags -> tags`
  - `explorer-categories -> categories`
  - `explorer-author -> author`
  - `explorer-archives -> archives`
