# 开发约束

## 结论

新增页面、组件、脚本时，默认遵守下面 6 条：

1. 页面只进 `src/apps/*`
2. 小组件只进 `src/widgets/*`
3. 公共壳层只进 `src/shell-core/*` 或 `src/shell/desktop-shell/*`
4. 协议字段只认单一来源
5. 新功能必须至少跑 `pnpm run check`
6. 不允许把旧 `features/*` 再引回来

## 目录约束

### 页面

新页面必须放到：

- [/src/apps](/src/apps)

页面目录最少包含：

- `entry.js`
- `manifest.js`
- `protocol.js`
- `state.js`
- `hydrate.js`

可选：

- `runtime/`
- `styles/`

### 小组件

新组件必须放到：

- [/src/widgets](/src/widgets)

组件目录最少包含：

- `manifest.js`
- `render.js`

### 公共壳层

只能放到：

- [/src/shell-core](/src/shell-core)
- [/src/shell/desktop-shell](/src/shell/desktop-shell)
- [/templates/modules/shell](/templates/modules/shell)

## import 约束

### 允许

- `apps/* -> shared/*`
- `apps/* -> shell-core 暴露接口`
- `apps/explorer/* -> apps/explorer/shared/*`

### 禁止

- `apps/* -> 其他 app 私有 runtime`
- `apps/* -> 旧 features runtime`
- `widgets/* -> 页面 DOM 私有结构`

## 协议约束

### Route

路由归类只能改：

- [/src/shell-core/runtime/route-manifest.js](/src/shell-core/runtime/route-manifest.js)

禁止：

- 在模板里写 app 判定正则
- 在 PJAX 里再写第二套路由归类

### App manifest

页面行为 contract 只能改：

- [/src/shell-core/runtime/app-manifests.js](/src/shell-core/runtime/app-manifests.js)
- [/src/apps](/src/apps)

禁止：

- 在 `pjax/index.js` 里重新维护 same-variant 白名单

### 页面协议

页面必须输出：

- `data-app-root="appId"`
- `data-app-props="appId"`

禁止：

- 散落业务状态到大量 `data-*`

## 标题栏与窗口约束

### 标题栏

只能通过这些片段扩展：

- [/templates/modules/shell/window-titlebar.html](/templates/modules/shell/window-titlebar.html)
- [/templates/modules/shell/window-titlebar-leading.html](/templates/modules/shell/window-titlebar-leading.html)
- [/templates/modules/shell/window-titlebar-actions.html](/templates/modules/shell/window-titlebar-actions.html)

禁止：

- 在页面模板里重新写一整套标题栏骨架

### 内容区

窗口内容容器只能走：

- [/templates/modules/shell/window-frame.html](/templates/modules/shell/window-frame.html)

禁止：

- 页面自己重新定义 `data-window-content-root`
- PJAX 替换窗口外壳

## 校验命令

开发时至少执行：

```bash
pnpm run check
```

可单独执行：

```bash
pnpm run lint
pnpm run typecheck
pnpm run smoke
pnpm run build-only
pnpm run smoke:playwright
```

浏览器级 smoke 需要提供站点地址：

```bash
SMOKE_BASE_URL="https://your-site.example.com" pnpm run smoke:playwright
```

可选详情页路径：

```bash
SMOKE_READER_PATH="/posts/demo"
SMOKE_AUTHOR_PATH="/author/demo"
SMOKE_MOMENT_DETAIL_PATH="/moments/demo"
```

## CI 约束

CI 在：

- [/.github/workflows/ci.yaml](/.github/workflows/ci.yaml)

当前默认执行：

- `pnpm run check`

如果仓库变量里提供了 `SMOKE_BASE_URL`，CI 还会额外执行 Playwright smoke。

## 不要再做的事

1. 不要把页面 runtime 再放回 `src/features/*`
2. 不要在模板中复制整套标题栏或窗口 body
3. 不要在 widget render 里直接依赖页面私有 DOM
4. 不要绕过 `asset-manifest.json` 手写资源索引
