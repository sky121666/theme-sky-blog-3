# 插件适配状态

## 判断标准

这里的“适配”按主题侧交付程度分三类：

1. 独立 App 适配：有页面模板、`src/apps/<app>` 入口、路由识别、窗口 variant、PJAX/缓存策略和移动端边界。
2. 组件级适配：主题只接入插件提供的组件或全局 API，不单独提供页面。
3. 未适配：没有主题模板、没有 app manifest，也没有纳入路由协议。

只创建了本地 skill 或调研文档，不等于主题已适配。

插件“适配了什么”看本文；插件“按哪个版本契约适配、哪些版本已经测试”看 [插件适配契约](插件适配契约.md)。

## 当前收口结论

当前已纳入主题的插件页面和组件适配已收口，没有已适配但契约版本未知的插件。

明确未适配项已清空；`plugin-douban` 已新增独立 App，契约确认到 `1.2.5`。

`plugin-forum` 已按用户决策不纳入本主题。

本轮已在 Halo `2.25.4` 读取精确插件 inventory，并把上游基线、安装版本、契约版本、已测试版本分层记录。Steam、Bangumi、Docsme、Moments、Photos、link-submit、Shiki、KaTeX、Text Diagram、SEO Tools、LightGallery 与 Online 已按各自专项证据更新；`PluginLinks` 按用户决定精确固定 `2.0.0`，不跟随 2.2.x。Comment Widget 3.1.2 资源已确认加载，但其提交/登录等状态矩阵仍按保守边界记录。

真实样本复验、性能采样和插件资源审计统一看 [验证与审计矩阵](验证与审计矩阵.md)。后续只在矩阵有证据时进入实现，不把“可增强”直接等同于“必须做”。
