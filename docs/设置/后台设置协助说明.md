# 后台设置协助说明

## 结论

这份文档给 **主题维护者、后端协助人员、站点管理员** 使用。
目标只有一个：明确后台该配什么、每组配置会影响哪里。

当前主题设置入口主要有两类：

1. 主题设置：[/settings.yaml](/settings.yaml)
2. 注解设置：[/annotation-setting.yaml](/annotation-setting.yaml)

## 常见需求怎么找

| 我想改什么 | 先看哪里 |
| --- | --- |
| 改顶栏标题、登录、搜索、时间 | `header` 组 |
| 改顶栏按钮图标 | `header.icons.*`，再看 [/docs/设置/图标与注解设置指南.md](/docs/设置/图标与注解设置指南.md) |
| 改桌面背景和配色 | `desktop.background.*`、`desktop.appearance.*` |
| 改桌面挂载图标 | `desktop.icons.*` |
| 改 Dock 菜单和交互参数 | `dock.*` |
| 改 Dock 图标 | 先看 [/annotation-setting.yaml](/annotation-setting.yaml) 中 `MenuItem` 注解 |
| 改瞬间封面和文案 | `moments.*` |
| 改豆瓣标题、说明、图标和配色 | `douban.*` |
| 改桌面小组件开关和天气参数 | `widgets.*` |
| 改 links 页面标题 | `links.profile.display_name` |
| 改友链分组图标 | 先看 [/annotation-setting.yaml](/annotation-setting.yaml) 中 `LinkGroup` 注解 |
| 改默认桌面布局 | `default_layout.layout_json`，并结合 [/docs/桌面小组件.md](/docs/桌面小组件.md) |
| 开调试日志 | `developer.debug_mode` |

## 主题设置分组

| 分组 | 作用 | 典型使用场景 |
| --- | --- | --- |
| `header` | 顶栏导航、标题、时间、登录、工具栏图标、二级菜单背景 | 调整顶栏展示和访客交互 |
| `desktop` | 桌面配色、背景、桌面图标来源 | 调整桌面视觉和挂载图标 |
| `widgets` | 小组件开关、移动端隐藏、天气参数 | 开启首页桌面层能力 |
| `dock` | Dock 数据源与外观交互参数 | 调整 Dock 图标和动画感受 |
| `moments` | 瞬间页封面图与名片文案 | 美化瞬间页顶部视觉 |
| `douban` | 豆瓣页标题、说明、图标和配色模式 | 调整 `/douban` 书影音归档页面 |
| `links` | links 页面标题 | 调整全部友链默认标题 |
| `default_layout` | 已发布默认桌面布局 JSON | 以服务端保存首页桌面默认布局 |
| `developer` | 调试模式 | 输出前端调试日志 |

## 各组设置说明

### 1. 顶栏

关键字段：

- `header.source.menu_name`
- `header.logo.icon`
- `header.logo.title`
- `header.theme.enable_frontend_setting`
- `header.theme.default_mode`
- `header.actions.search_enabled`
- `header.actions.auth_enabled`
- `header.actions.mobile_menu_enabled`
- `header.time.*`
- `header.auth.login_label`
- `header.icons.*`
- `header.dropdown.light_bg`
- `header.dropdown.dark_bg`

适合谁来配：

- 站点管理员
- 需要配品牌、菜单和访客入口的人

注意：

- 登录入口固定跳转 `/login`
- 搜索关闭后，`Cmd/Ctrl + K` 也会一起禁用
- 图标项留空时走主题内置 SVG

最常见需求：

- 想改品牌名称：看 `header.logo.title`
- 想改左上角图标：看 `header.logo.icon`
- 想改访客默认明暗模式：看 `header.theme.default_mode`
- 想隐藏登录入口：看 `header.actions.auth_enabled`

### 2. 桌面

关键字段：

- `desktop.appearance.*`
- `desktop.background.*`
- `desktop.icons.*`

适合谁来配：

- 需要调整桌面视觉的人
- 需要决定哪些分类、标签、文章、单页挂在桌面的人

注意：

- `desktop.icons.custom_icons` 适合自由挂载
- `categories / tags / posts / single_pages` 适合结构化内容挂载

最常见需求：

- 想换整套桌面配色：看 `desktop.appearance.mode/preset`
- 想自定义强调色：看 `desktop.appearance.accent_color`
- 想换桌面背景：看 `desktop.background.*`
- 想把单页挂到桌面：看 `desktop.icons.single_pages`

### 3. 小组件

关键字段：

- `widgets.behavior.enabled`
- `widgets.behavior.hide_on_mobile`
- `widgets.behavior.edit_enabled`
- `widgets.behavior.fallback_cover`
- `widgets.modules.weather.city_name`
- `widgets.modules.weather.refresh_minutes`

说明：

- 小组件固定在桌面层，不跟随 PJAX 页面切换消失
- 当前默认布局权威来源不是本地缓存，而是 `default_layout.layout_json`

最常见需求：

- 想彻底关掉小组件：看 `widgets.behavior.enabled`
- 想手机端隐藏：看 `widgets.behavior.hide_on_mobile`
- 想改天气默认城市：看 `widgets.modules.weather.city_name`

详见：

- [/docs/桌面小组件.md](/docs/桌面小组件.md)

### 4. Dock

关键字段：

- `dock.source.menu_name`
- `dock.appearance.show_labels`
- `dock.appearance.magnification`
- `dock.appearance.icon_size`
- `dock.appearance.icon_gap`
- `dock.appearance.dock_padding`
- `dock.appearance.magnification_scale`
- `dock.appearance.glass_blur`
- `dock.appearance.glass_opacity`

适合谁来配：

- 希望调整 Dock 交互手感的人

最常见需求：

- 想改 Dock 图标大小：看 `dock.appearance.icon_size`
- 想关掉放大效果：看 `dock.appearance.magnification`
- 想换 Dock 菜单来源：看 `dock.source.menu_name`

### 5. 瞬间

关键字段：

- `moments.cover.image_url`
- `moments.profile.display_name`
- `moments.profile.subtitle`

适合谁来配：

- 需要美化瞬间页封面的人

最常见需求：

- 想换瞬间封面图：看 `moments.cover.image_url`
- 想改封面显示名称：看 `moments.profile.display_name`
- 想改封面副标题：看 `moments.profile.subtitle`

### 6. 豆瓣

关键字段：

- `douban.profile.icon`
- `douban.profile.display_name`
- `douban.profile.subtitle`
- `douban.style.color_mode`

适合谁来配：

- 需要调整 `/douban` 页面识别和品牌感的人
- 需要在豆瓣绿与主题强调色之间切换的人

注意：

- 标题留空时使用当前页面标题
- 说明留空时使用默认书影音说明
- 图标留空时使用默认书影音图标
- 图标项使用 Iconify SVG 格式；如果传入的值不是合法 SVG，前端会保留默认图标
- `color_mode` 默认为 `douban`，可切换为 `theme` 跟随主题强调色

详见：

- [/docs/功能/豆瓣.md](/docs/功能/豆瓣.md)

### 7. 友链

关键字段：

- `links.profile.display_name`

说明：

- links 页面现在是独立窗口 app
- 这里配置的是“全部友链”默认标题
- 如果 URL 带了具体分组，窗口标题会优先显示当前分组名

最常见需求：

- 想改“全部友链”默认标题：看这里
- 想改分组图标：不要在主题设置里找，去看 `LinkGroup` 注解

详见：

- [/docs/功能/链接页.md](/docs/功能/链接页.md)

### 8. 默认布局

关键字段：

- `default_layout.layout_json`

说明：

- 这不是手工优先维护的字段
- 更推荐通过前端桌面编辑器发布
- 手工改时要保证 JSON 结构正确

最常见需求：

- 想全站替换默认桌面布局：看这里
- 想微调首页默认小组件位置：优先用前端编辑器，不建议直接手改 JSON

### 9. 开发者

关键字段：

- `developer.debug_mode`

说明：

- 开启后会在控制台输出 `[pjax]`、`[desktop]` 等调试日志
- 生产环境建议关闭

## 注解设置

当前注解设置来自：

- [/annotation-setting.yaml](/annotation-setting.yaml)

包括：

| 注解对象 | 用途 |
| --- | --- |
| `MenuItem` | Dock 图标来源、Header 子菜单分隔线 |
| `Tag` | 标签图标 |
| `Category` | 分类图标与颜色 |
| `PhotoGroup` | 图库相册图标与简介（用于侧边栏与小组件） |
| `LinkGroup` | 友链分组图标（用于 links 页侧边栏和分组标题） |

详见：

- [/docs/设置/图标与注解设置指南.md](/docs/设置/图标与注解设置指南.md)

## 后台协助建议

### 适合后台直接处理的内容

1. 顶栏菜单来源
2. 品牌名称和图标
3. 桌面背景与配色
4. Dock 菜单来源
5. 瞬间封面图
6. 桌面图标挂载项

### 不建议后台硬改的内容

1. `templates/assets` 下的构建产物
2. feature 目录和模板路径
3. PJAX 行为
4. shell 交互逻辑

这些属于代码层，不属于后台协助配置范围。
