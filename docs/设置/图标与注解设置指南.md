# 图标与注解设置指南

## 结论

当前主题的图标来源不是一条线，而是 4 条线：

1. Header 工具栏图标
2. 桌面图标
3. Dock 菜单项图标
4. 内容对象注解图标

搞不清楚时，先问自己：

- 这是页面级图标，还是内容对象图标？
- 这是后台设置，还是注解设置？

## 常见需求怎么找

| 我想改什么 | 先看哪里 |
| --- | --- |
| 改 Header 固定按钮图标 | `settings.yaml -> header.icons.*` |
| 改桌面挂载图标 | `settings.yaml -> desktop.icons.*` |
| 改 Dock 菜单项图标 | `annotation-setting.yaml -> MenuItem` |
| 改 Header 二级菜单前置分隔线 | `annotation-setting.yaml -> MenuItem.header_separator_before` |
| 改标签图标 | `annotation-setting.yaml -> Tag` |
| 改分类图标和颜色 | `annotation-setting.yaml -> Category` |
| 改图库相册图标 | `annotation-setting.yaml -> PhotoGroup` |
| 改图库相册简介（小组件底部说明文字） | `annotation-setting.yaml -> PhotoGroup` |
| 改友链分组图标 | `annotation-setting.yaml -> LinkGroup` |

## 1. Header 工具栏图标

配置位置：

- [/settings.yaml](/settings.yaml)

字段：

- `header.icons.mobile_menu`
- `header.icons.search`
- `header.icons.auth`
- `header.icons.theme_light`
- `header.icons.theme_dark`

规则：

- 留空：使用主题内置图标
- 非空：覆盖对应按钮图标
- 当前使用 `iconify(svg)` 字段类型

适用场景：

- 想统一替换顶部工具栏图标风格

## 2. 桌面图标

配置位置：

- [/settings.yaml](/settings.yaml)

字段：

- `desktop.icons.custom_icons`
- `desktop.icons.categories`
- `desktop.icons.tags`
- `desktop.icons.posts`
- `desktop.icons.single_pages`

### `custom_icons`

适合：

- 自定义挂载 `/about`
- 挂外链
- 想自己定图标风格

关键字段：

- `name`
- `href`
- `type`
- `external`

`type` 当前支持：

- `folder`
- `document`
- `link`

### 结构化挂载

适合：

- 分类
- 标签
- 多篇文章
- 单页

这几类不需要手写 href，直接从后台选择对象即可。

## 3. Dock 菜单项图标

配置位置：

- [/annotation-setting.yaml](/annotation-setting.yaml)

作用对象：

- `MenuItem`

支持 3 种来源：

1. `iconify`
2. `custom_svg`
3. `image`

对应字段：

- `icon_type`
- `icon`
- `custom_svg`
- `image_url`
- `bg_color`

建议：

- 默认优先 `iconify`
- 只有需要完全定制时才用 `custom_svg`
- 图片模式适合品牌图标或动态图片

额外字段：

- `header_separator_before`

作用：

- 只对 Header 的二级菜单项生效
- 会在当前项前面插入一条分隔线

## 4. 标签、分类、相册注解

### 标签

对象：

- `Tag`

字段：

- `icon`

用途：

- 随机标签小组件

### 分类

对象：

- `Category`

字段：

- `icon`
- `color`

用途：

- 桌面分类小组件
- 分类卡片视觉点缀

### 相册

对象：

- `PhotoGroup`

字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `icon` | iconify（SVG） | 相册图标，显示在侧边栏和图库小组件角标 |
| `description` | 文本 | 相册简介，显示在图库小组件底部说明区（建议 15 字以内） |

配置入口：

Halo 后台 → 图库 → 分组列表 → 编辑分组 → **元数据** 标签页

用途：

- `icon`：图库侧边栏相册图标 + 图库小组件角标 SVG
- `description`：图库小组件底部大字说明（优先于相册名显示）

> **注意**：若 `description` 留空，小组件底部大字回退显示相册 `displayName`。

### 友链分组

对象：

- `LinkGroup`

字段：

- `icon`

用途：

- links 页侧边栏分组图标
- links 页分组标题图标

配置入口：

Halo 后台 → 友链分组 → 编辑分组 → **元数据**

## 5. 选哪种配置链

| 需求 | 配置入口 |
| --- | --- |
| 改 Header 固定按钮图标 | `settings.yaml -> header.icons.*` |
| 改桌面挂载图标 | `settings.yaml -> desktop.icons.*` |
| 改 Dock 菜单项图标 | `annotation-setting.yaml -> MenuItem` |
| 改标签图标 | `annotation-setting.yaml -> Tag` |
| 改分类图标和颜色 | `annotation-setting.yaml -> Category` |
| 改图库相册图标 | `annotation-setting.yaml -> PhotoGroup.icon` |
| 改图库相册简介（小组件底部） | `annotation-setting.yaml -> PhotoGroup.description` |
| 改友链分组图标 | `annotation-setting.yaml -> LinkGroup.icon` |


## 6. 实用建议

1. 全站统一风格时，优先用 `iconify`
2. 想要品牌感时，Dock 单独用 `custom_svg` 或图片
3. 分类图标建议同时配颜色，否则图标信息量不够
4. Header 图标留空通常比乱配更稳，因为主题内置图标已经和壳层风格对齐

## 什么时候看这份文档

适合来看这份文档的场景：

1. 后台已经能配图标，但你不知道该配哪一组
2. 想区分 `settings.yaml` 和 `annotation-setting.yaml` 的职责
3. 想统一 Header、Dock、桌面、分类、标签、相册的图标来源
