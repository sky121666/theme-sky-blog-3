# 桌面小组件

## 结论

桌面小组件属于 **桌面层专题能力**，不是普通页面模块。
它同时涉及：

- 后台设置
- 默认布局协议
- 前端编辑器
- 运行时渲染

所以单独成文，不继续塞进架构总览里。

## 当前模型

### 渲染位置

- 只在首页桌面层渲染
- 不进入主窗口内容区
- 和桌面图标、Dock 同层工作

模板落点：

- [/templates/modules/shell/desktop-widgets.html](/templates/modules/shell/desktop-widgets.html)

### 权威数据来源

当前默认布局的权威来源是：

- `theme.config.default_layout.layout_json`

说明：

- 有值：按服务端发布布局渲染
- 为空：只显示桌面图标，不自动补默认小组件

### Runtime 入口

- [/src/shell/desktop-shell/runtime/desktop/surface/index.js](/src/shell/desktop-shell/runtime/desktop/surface/index.js)

职责：

- 布局解析
- 编辑态
- 拖拽与排布
- 小组件中心
- 保存发布

## 后台相关配置

### 直接配置

来自 [/settings.yaml](/settings.yaml)：

- `widgets.behavior.*`
- `widgets.modules.weather.*`
- `default_layout.layout_json`

### 核心开关

| 字段 | 作用 |
| --- | --- |
| `widgets.behavior.enabled` | 是否启用桌面小组件 |
| `widgets.behavior.hide_on_mobile` | 手机端是否隐藏 |
| `widgets.behavior.edit_enabled` | 是否允许进入编辑态 |
| `widgets.behavior.fallback_cover` | 无封面时的回退图 |
| `widgets.modules.weather.city_name` | 天气默认城市 |
| `widgets.modules.weather.refresh_minutes` | 天气刷新间隔 |
| `default_layout.layout_json` | 已发布默认布局 JSON |

## 默认布局协议

### 布局 JSON

服务端发布的默认布局结构核心是：

```json
{
  "version": 3,
  "layoutVersion": "v1",
  "instances": [],
  "icons": []
}
```

### `instances`

小组件实例最小结构：

```json
{
  "key": "clock-main",
  "title": "时间",
  "widget": "system.clock",
  "size": "small",
  "appearance": "follow",
  "x": 1,
  "y": 1
}
```

说明：

- `hidden` 只用于运行时，不进入服务端发布协议
- `w/h` 也是运行时推导，不作为后台权威字段
- `meta` 保存实例级配置，例如图库相册、文章数量、天气城市等

### 解析与写回

相关实现：

- [/src/shell/desktop-shell/runtime/widgets/persistence-read.js](/src/shell/desktop-shell/runtime/widgets/persistence-read.js)
- [/src/shell/desktop-shell/runtime/widgets/persistence-write.js](/src/shell/desktop-shell/runtime/widgets/persistence-write.js)

关键方法：

- `parseDesktopLayoutPayload()`
- `buildDesktopLayoutJsonString()`
- `applyDesktopLayoutJsonToThemeConfig()`

### 实例配置

支持配置的小组件会在 manifest 中声明：

- `hasConfig`
- `configSchema`
- `configDefaults`

当前支持实例配置的组件：

| 小组件 | 配置项 |
| --- | --- |
| `system.weather` | 城市、刷新间隔 |
| `halo.identity_card` | 是否显示 Steam、瞬间、文章、图库状态 |
| `halo.latest_posts` | 数量、摘要、分类 |
| `halo.popular_posts` | 数量、摘要 |
| `plugin-moments.recent` | 数量、媒体预览 |
| `plugin-photos.gallery` | 图库分组 |

优先级：

1. 实例 `meta` 优先，只影响当前小组件实例。
2. 后台 `widgets.modules.*` 是全局默认值。
3. manifest `configDefaults` 是缺省兜底。

说明：

- `meta` 写入 `default_layout.layout_json.instances[].meta`。
- 保存默认布局时，`meta` 会随实例一起回写。
- 组件中心预览模式不会触发 PJAX 跳转。

## 外观模式

每个小组件实例支持：

- `follow`
- `light`
- `dark`

规则：

- `follow`：跟随全站主题
- `light`：强制浅色
- `dark`：强制深色

样式核心落点：

- [/src/shell/desktop-shell/styles/widgets/base.css](/src/shell/desktop-shell/styles/widgets/base.css)
- [/src/shell/desktop-shell/styles/widgets/shell.css](/src/shell/desktop-shell/styles/widgets/shell.css)
- [/src/shell/desktop-shell/styles/widgets/content/index.css](/src/shell/desktop-shell/styles/widgets/content/index.css)

## 组件中心

当前组件中心负责：

- 搜索组件
- 分类浏览
- 选择尺寸
- 选择实例外观
- 添加组件
- 替换现有组件
- 保存布局

当前动作入口：

- `添加`
- `装饰`
- `推荐布局`
- `保存`

说明：

- `保存` 会把当前布局写回 `default_layout.layout_json`
- `推荐布局` 只修改本地编辑态布局，不会自动保存
- 无管理员权限时，不开放发布能力

## 新增组件时看哪里

### 1. 组件目录

先看：

- [/src/widgets/registry.js](/src/widgets/registry.js)
- [/src/widgets/catalog.js](/src/widgets/catalog.js)
- [/src/widgets/loaders.js](/src/widgets/loaders.js)

至少要注册：

- `title`
- `category`
- `size`
- `sizes`
- `description`

### 2. 渲染器

再看：

- [/src/widgets](/src/widgets)

至少要兼容：

- `small`
- `medium`
- `large`

实际支持哪些尺寸以各 widget manifest 的 `supportedSizes` 为准，不强制所有组件都实现三种尺寸。

### 3. 外观模式

组件根节点必须兼容：

- `data-widget-appearance`

不要再依赖：

- 本地布局缓存
- `pinMode`
- `visible` 写回服务端协议

### 4. 样式文件

内容样式统一从这里汇总：

- [/src/shell/desktop-shell/styles/widgets/content/index.css](/src/shell/desktop-shell/styles/widgets/content/index.css)

新增组件样式应放到同目录独立文件，并使用 `--widget-card-*`、`--theme-accent`、局部 token 收口颜色。

## 身份状态小组件（halo.identity_card）

身份状态卡不是独立 Steam 小组件，而是作者入口的增强版。

状态优先级：

1. Steam 明确返回 `profile.playing == true` 时显示正在游戏
2. 48 小时内有新瞬间时显示刚刚更新瞬间
3. 7 天内有新文章时显示最近写了文章
4. 图库可用且有照片/相册时显示最近整理图库
5. 以上都没有时回落到作者简介

边界：

- 只使用已确认来源：作者资料、`steamFinder`、瞬间列表、文章列表、图库列表。
- Steam 最近游玩列表只作为弱信息来源，不会被当成“正在游戏”。
- 插件缺失或字段为空时，卡片保持作者默认态，不输出负面空状态。
- 配置项写入实例 `meta`，可分别关闭 Steam、瞬间、文章、图库状态。

相关实现：

- [/src/widgets/halo/identity-card](/src/widgets/halo/identity-card)
- [/src/widgets/shared/presence.js](/src/widgets/shared/presence.js)
- [/scripts/verify-widgets-presence.mjs](/scripts/verify-widgets-presence.mjs)

## 瞬间小组件（plugin-moments.recent）

瞬间小组件是朋友圈/照片回忆入口，不在桌面阶段提前加载评论。

尺寸行为：

| 尺寸 | 内容 |
| --- | --- |
| `small` | 最新一条纯文本摘要，保留头像、时间和点赞数 |
| `medium` | 最新一条大卡，有媒体时显示一张预览图 |
| `large` | 最新一条作为主卡，下面追加后续两条列表 |

边界：

- 点击有 `metadata.name` 的瞬间进入 `/moments/{name}`，无内容时进入 `/moments`。
- 只显示 `stats.upvote` 点赞数；评论数量不在小组件里显示，避免和列表/详情评论加载策略冲突。
- 不请求评论 API，不加载全部评论；评论仍由 Moments 列表页预览和详情页完整评论负责。
- 配置项只影响大尺寸最多展示条数，以及是否显示媒体预览。

## 朋友圈小组件（plugin-friends.recent）

朋友圈小组件按 `plugin-friends` 1.4.5 的 `friendFinder.list(1, 5)` 契约读取最近友链动态。

尺寸行为：

| 尺寸 | 内容 |
| --- | --- |
| `medium` | 最新 2-3 条友链动态 + 朋友圈入口 |
| `large` | 最新 1 条主动态 + 后续 4 条列表 + 朋友圈入口 |

边界：

- 只使用插件 Finder 注入的 `FriendPostVo`，不解析 `/friends` 页面 DOM。
- 动态卡片点击打开原文外链，底部入口进入主题 `/friends` 页面。
- 插件未安装时不会出现在组件中心；插件安装但暂无数据时显示空态。

## 文档小组件（plugin-docsme.quick）

文档小组件按 Docsme `1.6.0` 页面契约提供 `/docs` 快捷入口，不读取私有文档树或未确认接口。

尺寸行为：

| 尺寸 | 内容 |
| --- | --- |
| `small` | 文档中心图标 + Docsme 标记 |
| `medium` | 文档中心说明 + `/docs` 打开按钮 |

边界：

- 只在 `plugin-docsme` 可用时出现在组件中心。
- 点击进入主题 Docsme App 的 `/docs` 项目大厅，并保留 PJAX。
- 不调用 `docsmeProjectsFinder`，后续如需要项目数量再单独做版本门控和权限复验。

## 图库小组件（plugin-photos.gallery）

### 功能说明

需要安装 Halo 官方图库插件（`plugin-photos`）后，图库小组件才会出现在组件中心。

它能在桌面轮播展示你的照片。支持 3 种尺寸：

| 尺寸 | 效果 |
| --- | --- |
| `small` (2×2) | 封面图铺满，顶部胶囊（icon + 相册名），底部简介 + 张数 |
| `medium` (4×2) | 封面图铺满，右上角胶囊，底部大字简介 + 张数 |
| `large` (4×4) | 封面图铺满，左上图标框 + 相册名，底部大字简介 + 缩略图行 |

### 如何设置相册图标与简介

小组件展示时，会读取相册自带的“图标”与“简介”来丰富卡片信息。

**操作入口：**
Halo 后台 → 图库 → 分组列表 → 编辑分组 → **元数据** 标签页

| 字段（标签名） | 说明 |
| --- | --- |
| `icon` | 相册图标（SVG代码），显示在顶部角标或图标框内 |
| `description` | 相册一句话简介（建议15字内），显示在底部大字主说明区 |

> 提示：如果 `description` 留空，小组件底部会自动降级显示相册本身的名称。

### 如何添加到桌面

1. 首页开启桌面编辑，打开 **组件中心**
2. 找到 **图库** 并点击添加
3. 弹出配置框，从相册列表中选择一个相册（**必须选择**，不支持展示全部）
4. 选择需要的尺寸，点击 **添加到桌面** 并保存。

> 提示：如果后台图库还没有创建任何分组，配置弹窗会提示「还没有相册」，请先在 Halo 后台图库中创建分组后再添加小组件。
