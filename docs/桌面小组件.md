# 桌面小组件

## 结论

桌面小组件属于 **桌面层专题能力**，不是普通页面模块。
它同时涉及：

- 后台设置
- 默认布局协议
- 前端编辑器
- 运行时渲染

所以单独成文，不继续塞进架构总览里。

## 当前模型

### 渲染位置

- 只在首页桌面层渲染
- 不进入主窗口内容区
- 和桌面图标、Dock 同层工作

模板落点：

- [/templates/modules/shell/desktop-widgets.html](/templates/modules/shell/desktop-widgets.html)

### 权威数据来源

当前默认布局的权威来源是：

- `theme.config.default_layout.layout_json`

说明：

- 有值：按服务端发布布局渲染
- 为空：只显示桌面图标，不自动补默认小组件

### Runtime 入口

- [/src/shell/desktop-shell/runtime/desktop/surface/index.js](/src/shell/desktop-shell/runtime/desktop/surface/index.js)

职责：

- 布局解析
- 编辑态
- 拖拽与排布
- 小组件中心
- 保存发布

## 后台相关配置

### 直接配置

来自 [/settings.yaml](/settings.yaml)：

- `widgets.behavior.*`
- `widgets.modules.weather.*`
- `default_layout.layout_json`

### 核心开关

| 字段 | 作用 |
| --- | --- |
| `widgets.behavior.enabled` | 是否启用桌面小组件 |
| `widgets.behavior.hide_on_mobile` | 手机端是否隐藏 |
| `widgets.behavior.edit_enabled` | 是否允许进入编辑态 |
| `widgets.behavior.fallback_cover` | 无封面时的回退图 |
| `widgets.modules.weather.city_name` | 天气默认城市 |
| `widgets.modules.weather.refresh_minutes` | 天气刷新间隔 |
| `default_layout.layout_json` | 已发布默认布局 JSON |

## 默认布局协议

### 布局 JSON

服务端发布的默认布局结构核心是：

```json
{
  "version": 3,
  "layoutVersion": "v1",
  "instances": [],
  "icons": []
}
```

### `instances`

小组件实例最小结构：

```json
{
  "key": "clock-main",
  "title": "时间",
  "widget": "system.clock",
  "size": "small",
  "appearance": "follow",
  "x": 1,
  "y": 1
}
```

说明：

- `hidden` 只用于运行时，不进入服务端发布协议
- `w/h` 也是运行时推导，不作为后台权威字段
- `meta` 保存实例级配置，例如图库相册、文章分类、天气城市等

### 解析与写回

相关实现：

- [/src/shell/desktop-shell/runtime/widgets/persistence-read.js](/src/shell/desktop-shell/runtime/widgets/persistence-read.js)
- [/src/shell/desktop-shell/runtime/widgets/persistence-write.js](/src/shell/desktop-shell/runtime/widgets/persistence-write.js)

关键方法：

- `parseDesktopLayoutPayload()`
- `buildDesktopLayoutJsonString()`
- `applyDesktopLayoutJsonToThemeConfig()`

### 实例配置

支持配置的小组件会在 manifest 中声明：

- `hasConfig`
- `configSchema`
- `configDefaults`

当前支持实例配置的组件：

| 小组件 | 配置项 |
| --- | --- |
| `system.weather` | 城市、刷新间隔 |
| `halo.latest_posts` | 分类 |
| `halo.categories` | 最多 4 个展示分类 |
| `plugin-photos.gallery` | 图库分组 |
| `plugin-douban.showcase` | 类型、状态 |

优先级：

1. 实例 `meta` 优先，只影响当前小组件实例。
2. 后台 `widgets.modules.*` 是全局默认值。
3. manifest `configDefaults` 是缺省兜底。

说明：

- `meta` 写入 `default_layout.layout_json.instances[].meta`。
- 保存默认布局时，`meta` 会随实例一起回写。
- 组件中心预览模式不会触发 PJAX 跳转。
- 固定版式组件不暴露数量、摘要、媒体开关这类视觉配置；每个尺寸按组件自身设计决定展示条数和信息密度。

## 外观模式

每个小组件实例支持：

- `follow`
- `light`
- `dark`

规则：

- `follow`：跟随全站主题
- `light`：强制浅色
- `dark`：强制深色

样式核心落点：

- [/src/shell/desktop-shell/styles/widgets/base.css](/src/shell/desktop-shell/styles/widgets/base.css)
- [/src/shell/desktop-shell/styles/widgets/shell.css](/src/shell/desktop-shell/styles/widgets/shell.css)
- [/src/shell/desktop-shell/styles/widgets/content/index.css](/src/shell/desktop-shell/styles/widgets/content/index.css)

## 组件中心

当前组件中心负责：

- 搜索组件
- 分类浏览
- 选择尺寸
- 选择实例外观
- 添加组件
- 替换现有组件
- 保存布局

当前动作入口：

- `添加`
- `装饰`
- `推荐布局`
- `保存`

说明：

- `保存` 会把当前布局写回 `default_layout.layout_json`
- `推荐布局` 只修改本地编辑态布局，不会自动保存
- 无管理员权限时，不开放发布能力

## 当前组件清单

| 分类 | 小组件 |
| --- | --- |
| 系统 | `system.clock`、`system.calendar`、`system.weather` |
| Halo | `halo.latest_posts`、`halo.popular_posts`、`halo.categories`、`halo.author_card`、`halo.site_stats`、`halo.random_tags` |
| 插件 | `plugin-moments.recent`、`plugin-bangumis.recent`、`plugin-photos.gallery`、`plugin-douban.showcase`、`plugin-friends.recent`、`plugin-docsme.quick` |

插件组件只在对应插件可用时进入组件中心。缺插件时不展示入口，不让组件中心报错。

### 豆瓣收藏组件

`plugin-douban.showcase` 只提供大尺寸卡片，用于展示书影音收藏精选。

数据来源：

- 插件可用性：`pluginFinder.available('plugin-douban')`
- 类型列表：`/apis/api.douban.moony.la/v1alpha1/doubanmovies/-/types`
- 条目列表：`/apis/api.douban.moony.la/v1alpha1/doubanmovies`

实例配置：

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `type` | `auto` | `auto`、`movie`、`book`、`music`、`game`、`drama` |
| `status` | `auto` | `auto`、`all`、`mark`、`doing`、`done` |

`status` 只作为数据筛选条件；小组件卡片本身不展示“想看 / 在看 / 看过”等状态统计，避免大卡片信息过密。

交互规则：

- 自动轮播当前集合。
- 鼠标悬停或键盘聚焦时暂停轮播。
- 悬停底部缩略条目时切换当前展示。
- `prefers-reduced-motion: reduce` 下不自动轮播。

## 新增组件时看哪里

### 1. 组件目录

先看：

- [/src/widgets/registry.js](/src/widgets/registry.js)
- [/src/widgets/catalog.js](/src/widgets/catalog.js)
- [/src/widgets/loaders.js](/src/widgets/loaders.js)

至少要注册：

- `title`
- `category`
- `size`
- `sizes`
- `description`

### 2. 渲染器

再看：

- [/src/widgets](/src/widgets)

至少要兼容：

- `small`
- `medium`
- `large`

实际支持哪些尺寸以各 widget manifest 的 `supportedSizes` 为准，不强制所有组件都实现三种尺寸。

### 3. 外观模式

组件根节点必须兼容：

- `data-widget-appearance`

不要再依赖：

- 本地布局缓存
- `pinMode`
- `visible` 写回服务端协议

### 4. 样式文件

内容样式统一从这里汇总：

- [/src/shell/desktop-shell/styles/widgets/content/index.css](/src/shell/desktop-shell/styles/widgets/content/index.css)

新增组件样式应放到同目录独立文件，并使用 `--widget-card-*`、`--theme-accent`、局部 token 收口颜色。

## 瞬间小组件（plugin-moments.recent）

瞬间小组件是朋友圈/照片回忆入口，不在桌面阶段提前加载评论。

尺寸行为：

| 尺寸 | 内容 |
| --- | --- |
| `small` | 最新一条沉浸式方形卡；有媒体时首图铺底，叠加头像、时间、摘要、点赞和评论数 |
| `medium` | 最新一条横向卡；左侧作者/摘要/标签/点赞/评论数，右侧首个媒体预览 |
| `large` | 最新一条大卡；顶部作者信息，中部摘要和大图，底部标签、点赞与评论数 |

边界：

- 点击有 `metadata.name` 的瞬间进入 `/moments/{name}`，无内容时进入 `/moments`。
- 作者信息优先使用当前瞬间的 `owner.avatar` / `owner.displayName`，缺失时回退到站点作者资料。
- 显示 `stats.upvote` 点赞数和评论统计；评论数优先使用 `stats.approvedComment`，缺失时兜底 `stats.totalComment`。
- 不请求评论 API，不加载全部评论；评论内容仍由 Moments 列表页预览和详情页完整评论负责。
- 不提供组件配置项；数量固定为最新一条，媒体预览是该组件核心视觉，不做开关。

## 朋友圈小组件（plugin-friends.recent）

朋友圈小组件按 `plugin-friends` 1.4.5 的 `friendFinder.list(1, 5)` 契约读取最近友链动态。

尺寸行为：

| 尺寸 | 内容 |
| --- | --- |
| `small` | 最新 1 条沉浸式暗调方形卡；使用来源 logo 做模糊背景，叠加标题、时间和头像 |
| `medium` | 最新 1 条横向沉浸卡；显示标题、摘要、来源头像、来源名、`linkName` 和时间 |
| `large` | 最新 1 条主动态 + 后续 3 条列表；悬停列表时切换对应来源 logo 模糊背景 |

边界：

- 只使用插件 Finder 注入的 `FriendPostVo`，不解析 `/friends` 页面 DOM。
- 动态卡片点击打开原文外链；不额外插入入口按钮，避免破坏沉浸式固定版式。
- 可用字段限定为 `metadata.name`、`spec.title`、`spec.description`、`spec.author`、`spec.logo`、`spec.postLink`、`spec.authorUrl`、`spec.linkName`、`spec.pubDate`；不伪造点赞、评论、媒体预览。
- 插件未安装时不会出现在组件中心；插件安装但暂无数据时显示空态。

## 文档小组件（plugin-docsme.quick）

文档小组件按 Docsme `1.6.0` 页面契约和 `docsmeProjectsFinder` 公共项目参数展示文档项目。Finder 能力最低需要 `plugin-docsme >= 1.4.0`。

尺寸行为：

| 尺寸 | 内容 |
| --- | --- |
| `small` | 一个项目入口或文档中心总览；显示图标、标题、项目/文档数量 |
| `medium` | 文档中心摘要 + 主项目 + 最多 2 个项目简表 |
| `large` | 文档中心摘要 + 主项目详情 + 最多 4 个项目列表 + 项目/文档总数 |

边界：

- 只在 `plugin-docsme >= 1.4.0` 可用时出现在组件中心。
- 数据来自 `docsmeProjectsFinder.list(1, 5)`，不读取私有文档树或页面 DOM。
- 可用字段限定为 `metadata.name`、`spec.displayName`、`spec.description`、`spec.icon`、`spec.authorizeRequired`、`spec.enableComment`、`status.permalink`、`status.totalDocs`。
- `status.totalDocs <= 0` 的项目不生成可点击详情入口，只显示禁用态。
- 点击项目进入插件生成的 `status.permalink`，点击“查看全部”进入 `/docs`，并保留 PJAX。

## 图库小组件（plugin-photos.gallery）

### 功能说明

需要安装 Halo 官方图库插件（`plugin-photos`）后，图库小组件才会出现在组件中心。

它能在桌面轮播展示你的照片。支持 3 种尺寸：

| 尺寸 | 效果 |
| --- | --- |
| `small` (2×2) | 封面图铺满，顶部胶囊（icon + 相册名），底部简介 + 张数 |
| `medium` (4×2) | 封面图铺满，右上角胶囊，底部大字简介 + 张数 |
| `large` (4×4) | 封面图铺满，左上图标框 + 相册名，底部大字简介 + 缩略图行 |

### 如何设置相册图标与简介

小组件展示时，会读取相册自带的“图标”与“简介”来丰富卡片信息。

**操作入口：**
Halo 后台 → 图库 → 分组列表 → 编辑分组 → **元数据** 标签页

| 字段（标签名） | 说明 |
| --- | --- |
| `icon` | 相册图标（SVG代码），显示在顶部角标或图标框内 |
| `description` | 相册一句话简介（建议15字内），显示在底部大字主说明区 |

> 提示：如果 `description` 留空，小组件底部会自动降级显示相册本身的名称。

### 如何添加到桌面

1. 首页开启桌面编辑，打开 **组件中心**
2. 找到 **图库** 并点击添加
3. 弹出配置框，从相册列表中选择一个相册（**必须选择**，不支持展示全部）
4. 选择需要的尺寸，点击 **添加到桌面** 并保存。

> 提示：如果后台图库还没有创建任何分组，配置弹窗会提示「还没有相册」，请先在 Halo 后台图库中创建分组后再添加小组件。
